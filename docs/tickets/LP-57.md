## Parser Hardening (the heart)

### Which files were used — HONEST statement

- **Real files:** exactly **one** — `MISMO16940192.xml` (a Conventional/Purchase loan, single
  borrower). Checked `/mnt/user-data/uploads/` and the whole repo: **no additional real MISMO files
  were supplied.**
- **Synthetic variants** (derived from that one real file via `tests/mismo/synthetic.py`, varying one
  thing at a time so tests stay anchored on the real shape):
  - **FHA** — `MortgageType=FHA` (V1 supports Conventional + FHA).
  - **Unsupported type** — `MortgageType=VA` (out of V1 scope; must tolerate, not crash).
  - **Multi-borrower** — a genuine **distinct** second borrower (different name + income +
    classification), so attribution can be verified, not just a duplicate.
  - **Missing optional sections** — liabilities, assets, and the property value dropped entirely.
  - **Zero-income deal** — all income items removed.
  - **HTML-wrapped** — the XML wrapped in an HTML island.

### What variation was confirmed

- **Multi-borrower attribution is correct** — each borrower keeps their own income/employers (the
  co-borrower's `3333.00` income and the primary's `7000/9400` do **not** cross-bleed), through both
  the parser and the import service. This was the main real risk; LP-51's per-party scoping holds.
- **FHA / VA / unknown mortgage types are tolerated** — read verbatim; the importer maps Conv/FHA to
  the program enum and leaves unknowns `None` (file still created).
- **Missing sections degrade gracefully** — dropped sections → empty lists (not errors); the rest of
  the deal still parses exactly; the needed-now property-value warning fires.
- **HTML-wrapped** parses identically (`source_format == "html"`).

### What the parser was FIXED / changed

- **No fix was required** — no real second file exposed a gap, and the synthetic variants confirmed
  the existing tolerance. (Honest: synthetic variants can't reveal the LOS-specific gaps a real file
  would.)
- **One proactive hardening added** — a non-blocking needed-now warning when **no borrower has any
  income** ("No income was found for any borrower."). Income drives DTI, so a silent zero-income parse
  is almost always an incomplete file or a parse gap. It does **not** fire on the real fixture (which
  has income), so existing exact-value tests are unchanged.

### HONEST robustness statement — what's still needed

The parser is validated against **one real file + synthetic variants of it**. This proves tolerance to
the *specific* structural variations applied; it does **not** prove robustness against real-world LOS
variation (different element ordering/namespaces, FHA-specific sections like UFMIP / MIP / FHA case
number, true co-borrower layouts, other LOS exporters). **To fully harden, supply real files —
especially a real FHA file and a real multi-borrower (co-borrower) export** — drop them into
`backend/tests/fixtures/mismo/`, parse + import each, and fix any XPath/tolerance gap they reveal
(targeted, not a redesign). The synthetic builder and the full-flow/isolation tests are the harness
they slot into. **We do not claim robustness beyond what these files demonstrate.**

## Full-Flow Integration (the journey tested)

`test_mismo_flow.py`, real stack (HTTP + DB + parsing + storage; only AI/Celery mocked, and MISMO uses
neither):

1. **Upload** the real fixture → `201`, a populated `LoanFile` (correct borrower/loan/program/purpose).
2. **Stored + readable** — the stated financials (income/employers/liabilities/assets + extended core)
   read back via the LP-55 endpoint.
3. **Edit** via LP-56 — update a stated value, add a row, remove a row (soft delete), edit a core field
   (note rate) → all persist; the read reflects them; an activity is logged (import + edits).
4. **HTML-wrapped** upload → same populated result (`source_format == "html"`).
5. **Malformed / non-MISMO** upload → graceful LP-46 envelope (`400`, `error.type == "bad_request"`),
   never a 500, no internals leaked.
6. **Hardening variants import correctly** — FHA → `loan_program == "fha"`; multi-borrower → 2
   borrowers with correct per-borrower income; zero-income → created with the warning.

## Tenant-Isolation Pass (security-critical) — enumerated new MISMO endpoints

Company A imports a file; company B is denied on **each** endpoint with **404** (anti-enumeration —
not 403):

| Endpoint | Method | LP | Cross-company |
|---|---|---|---|
| `/loan-files/{id}/stated-financials` | GET | 55 | 404 |
| `/loan-files/{id}/stated-liabilities` | POST | 56 | 404 |
| `/stated-liabilities/{id}` | PATCH / DELETE | 56 | 404 |
| `/loan-files/{id}/stated-assets` | POST | 56 | 404 |
| `/stated-assets/{id}` | PATCH / DELETE | 56 | 404 |
| `/loan-files/{id}/borrowers/{bid}/stated-income` | POST | 56 | 404 |
| `/stated-income-items/{id}` | PATCH / DELETE | 56 | 404 |
| `/loan-files/{id}/borrowers/{bid}/stated-employers` | POST | 56 | 404 |
| `/stated-employers/{id}` | PATCH / DELETE | 56 | 404 |
| `/loan-files/{id}` (note rate, terms) | PATCH | 56 (reused Epic-4) | 404 |
| `/loan-files/{id}/property` (valuation, …) | PATCH | 56 (reused Epic-4) | 404 |
| `/loan-files/{id}/borrowers/{bid}` (DOB, citizenship, SSN replace) | PATCH | 56 (reused Epic-4) | 404 |
| `/loan-files/import-mismo` | POST | 54 | creates for caller's OWN company; the created file is unreachable by another tenant (asserted) |

**PII / SSN:** on import, read, and edit responses, the raw SSN never appears; only `masked_ssn`
(e.g. `***-**-2233`) is returned, and there is no raw `ssn` key. An SSN replace echoes only the new
mask, never the raw value.

## Polish (MISMO-flow rough edges found + fixed)

A dry-run of the flow (upload → import-directly → display → edit) found the components already
consistent with LP-46/47 (friendly safe-error toasts, `aria-busy`, spinners, disabled-while-pending,
honest non-blocking warnings). One genuine rough edge fixed:

- **Editor empty-group states** — in edit mode, a stated group with no rows (e.g. a file with no
  assets) previously showed a bare header + Add. It now shows a dashed "None imported — use “Add” if
  any apply." hint, so an empty group reads as intentional, not broken. Applied to income/employers,
  liabilities, and assets.

The upload surface, new-file page, warnings banner, and error/loading states were reviewed and left
as-is (already polished) — no churn for its own sake.

## Seed Update (the MISMO-imported file, synthetic PII)

`seed_dev_data.py` seeds a **4th loan file imported from MISMO**: the real fixture is read and
**scrubbed to fully-synthetic PII** (`_synthetic_mismo_bytes` — every individual name → "Casey Demo",
email → `casey.demo@example.com`, phone → synthetic, **every SSN → `900223344`**, a never-issued 900-
value), then run through the **real** `create_loan_file_from_mismo` (LP-53). So the seed includes a
file with real stated financials (income/employers/liabilities/assets), the stored raw file, and a
`MismoImport` record — the MISMO path end to end.

- **Synthetic PII verified** — the seeded borrower is "Casey Demo" / `***-**-3344`; the stored raw file
  contains no real name/SSN/email (no "Mahesh"/"Chhotala"/`900112233`/`maheshmpc9`).
- **Idempotent + production-guarded** — inside the existing check-and-skip (`_seed_loan_files` skips if
  the company already has files); production guard unchanged; gracefully skipped (logged) if the fixture
  is absent.

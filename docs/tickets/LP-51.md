# LP-51 — MISMO Parsing Core

- **Ticket:** LP-51 — MISMO parsing core (deterministic, tolerant, typed-core + catch-all)
- **Epic:** Phase 1.5 — MISMO Import
- **Status:** Completed
- **Date:** 2026-06-12

## Summary

Opens Phase 1.5 (MISMO import — the **primary** file-creation path: the sister receives loan
applications as MISMO 3.4 XML from her LOS, not by typing forms). This ticket is the deterministic
**parsing core**: `parse_mismo(content) -> ParsedMismo` turns a MISMO XML (or HTML-wrapped XML) file
into a structured **typed core + catch-all** intermediate representation — tolerant of structural
variation, exact on values, validated, and tested against a **real** sample file. Backend only; no
AI; no frontend. Mapping-to-models, file creation, the upload endpoint, and the UI build on this.

## What Was Built

- **`app/mismo/schema.py`** — the parsed-data shape (Pydantic): `ParsedMismo` (borrowers, loan,
  property, liabilities, assets, catch_all, parse_warnings, source_format) + `ParsedBorrower`
  (name/SSN/DOB/marital/dependents/classification/contact/address/citizenship + `income_items`,
  `employers`, `declarations`), `ParsedIncomeItem`, `ParsedLoan`, `ParsedProperty`,
  `ParsedLiability`, `ParsedAsset`, `CatchAllSection`/`CatchAllField`. Money/rates are `Decimal`,
  dates are `date`; every field is optional (missing → `None`/`[]`).
- **`app/mismo/parser.py`** — `parse_mismo`, the per-entity parse helpers (the real namespace map +
  paths), `_extract_xml` (HTML-wrapped handling), `MismoParseError`, tolerant coercion
  (`_to_decimal`/`_to_date`/`_to_int`/`_to_bool`), and the catch-all walk. XXE-safe lxml parser
  (`resolve_entities=False`, `no_network=True`). Metadata-only logging.
- **`tests/fixtures/mismo/MISMO16940192.xml`** — the real sample committed as a fixture.
- **`tests/mismo/test_parser.py`** — 14 tests (exact-value, catch-all, multi-borrower, tolerance,
  HTML-wrapped, malformed/not-MISMO, no-SSN-logged).
- **lxml** added (`uv add lxml`).

## The Real File Structure

`MESSAGE → DEAL_SETS → DEAL_SET → DEALS → DEAL`. Under `DEAL`: `ASSETS`, `COLLATERALS`,
`LIABILITIES`, `LOANS`, `PARTIES`, `RELATIONSHIPS`. Namespaces: MISMO default
(`http://www.mismo.org/residential/2009/schemas`, prefix `m`) + `DU`/`ULAD`/`xlink`.

Typed-core paths (relative to `DEAL`, MISMO default ns):

| Entity | Path (key fields) |
|---|---|
| Borrower | `PARTIES/PARTY` where `ROLES/ROLE/ROLE_DETAIL/PartyRoleType == "Borrower"`; name `INDIVIDUAL/NAME/*`; DOB/marital/dependents/classification under `ROLE/BORROWER/BORROWER_DETAIL`; SSN via `TAXPAYER_IDENTIFIER[TaxpayerIdentifierType="SocialSecurityNumber"]/TaxpayerIdentifierValue`; contact under `INDIVIDUAL/CONTACT_POINTS`; `ADDRESSES/ADDRESS`; citizenship under `DECLARATION_DETAIL/CitizenshipResidencyType` |
| Income | `ROLE/BORROWER/CURRENT_INCOME/CURRENT_INCOME_ITEMS/CURRENT_INCOME_ITEM/CURRENT_INCOME_ITEM_DETAIL` → `CurrentIncomeMonthlyTotalAmount`, `IncomeType`, `EmploymentIncomeIndicator` |
| Employers | `ROLE/BORROWER/EMPLOYERS/EMPLOYER//FullName` |
| Declarations | `ROLE/BORROWER/DECLARATION/DECLARATION_DETAIL/*` (all indicators) |
| Loan | `LOANS/LOAN/TERMS_OF_LOAN/{BaseLoanAmount,NoteAmount,NoteRatePercent,LoanPurposeType,MortgageType,LienPriorityType}`; `AMORTIZATION/AMORTIZATION_RULE/AmortizationType`; `.//LoanAmortizationPeriodCount`; `LOAN_DETAIL/ApplicationReceivedDate` |
| Property | `COLLATERALS/COLLATERAL/SUBJECT_PROPERTY` → `ADDRESS/*`, `.//PropertyEstimatedValueAmount`, `.//PropertyValuationAmount`, `.//SalesContractAmount`, `PropertyUsageType`, `AttachmentType`, `ConstructionMethodType`, `FinancedUnitCount` |
| Liabilities | `LIABILITIES/LIABILITY` → `LIABILITY_DETAIL/{LiabilityType,LiabilityMonthlyPaymentAmount,LiabilityUnpaidBalanceAmount}`; `LIABILITY_HOLDER//FullName` |
| Assets | `ASSETS/ASSET` → `ASSET_DETAIL/{AssetType,AssetCashOrMarketValueAmount}`; `ASSET_HOLDER//FullName` |

> Note: `LoanPurposeType`/`MortgageType`/`LienPriorityType` live under `TERMS_OF_LOAN` (verified
> against the file), not as separate siblings.

## Why Deterministic, Not AI

MISMO is a standardized, machine-parseable schema and the LOS emits compliant MISMO, so lxml/XPath
is exact, free, fast, and auditable. The stated financial data is the **source-of-truth baseline**
(the stated side of stated-vs-verified) and must be read exactly — an AI misread of stated income or
loan amount would corrupt that baseline. **AI-fallback** for non-compliant files is a documented
**future** option, not built here.

## Typed Core + Catch-All (nothing dropped)

The typed core is a curated view; the catch-all captures **every other leaf** in the deal, grouped
by its section (the local-name path from `DEAL` to the leaf's parent). The parser tracks which
leaves the typed core consumed (by stable element path), so the catch-all is exactly "everything
else." On the real file: **45 catch-all sections**, including FIPS codes, counseling events, and
owned-property details — nothing is lost. The **SSN is consumed** by the typed core, so it never
appears in the catch-all.

## Tolerance + Validation + Graceful Errors

- **Tolerant:** a missing/optional element → `None`/`[]`; for needed-now fields (no borrower, no
  loan / missing base amount, no property / missing estimated value, a borrower with no name) a
  `parse_warning` is added. Never crashes on a missing element. Multiple borrowers supported
  (find-all on Borrower parties).
- **Validation (raises `MismoParseError`, safe message):** not valid XML → "File is not valid XML";
  valid XML but not a MISMO `MESSAGE` → "File is not a MISMO message"; MESSAGE with no `DEAL` →
  "MISMO file has no DEAL". MISMO-but-missing-data does **not** raise — it returns a partial parse
  with warnings.

## PII

The borrower SSN is parsed into `ParsedBorrower.ssn` (so downstream can encrypt/mask it) but is
**never logged**. Logging is **metadata-only** — `source_format`, counts (borrowers/liabilities/
assets/catch-all sections), warning count — never the SSN, names, amounts, or raw content. The SSN
is also kept out of the catch-all. (Encryption/masking is the downstream mapping/model ticket.)

- **Tolerance**: stripping `PropertyEstimatedValueAmount` → `estimated_value is None` + a warning,
  rest still parses, no crash.

- **Errors**: non-XML → `MismoParseError`; valid-XML-not-MISMO → "…MISMO…"; MESSAGE-without-DEAL →
  error.
- **Privacy**: captured logs contain neither the SSN, the name, nor amounts; the `mismo_parsed`
  metadata event is present.

## Decisions Made

- **ADR-159** — Deterministic MISMO parsing (lxml/XPath): typed core + catch-all, tolerant, exact,
  HTML-wrapped, validated, no-PII logging.

## Assumptions

- Mapping `ParsedMismo` → DB models + file creation (and SSN encryption) is the **next** ticket;
  this produces the intermediate representation only.
- AI-fallback for non-compliant files is a future option (not built).
- Multi-borrower is supported; re-import/versioning/diff is a later concern.
- The typed core grows as later phases (verification) need more fields.

## Notes

- Re-supply more real MISMO files later to harden tolerance against bounded schema variation.
- The catch-all's section keys are local-name paths; instances of the same section collapse into one
  section (label/value pairs) — acceptable for a "nothing dropped" bag.

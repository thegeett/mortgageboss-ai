# LP-37 — Anthropic Async Client Wrapper

- **Ticket:** LP-37 — Anthropic async client wrapper (singleton, transient retries, metadata logging, cost estimation)
- **Epic:** Epic 5 — Document Upload & Processing
- **Status:** Completed
- **Date:** 2026-06-11

## Summary

Built the **single gateway** for every Claude API call. It owns the cross-cutting concerns
— a shared singleton client, transient-only retries, metadata-only logging, latency timing,
and token-usage surfacing — so the AI features (classification LP-38, extraction LP-39) call
`complete(...)` and focus on their own logic. First AI-touching ticket; backend only; tests
fully mock the SDK (no real calls, no key needed).

## What Was Built

- **Singleton client** (`app/ai/client.py::get_anthropic_client`) — a lazy, `lru_cache`d
  `AsyncAnthropic` built from `settings.anthropic_api_key` (LP-35 factory style). The
  missing-key error (`AIClientError`) fires at **call time**, not import. SDK retries are
  disabled (`max_retries=0`) so the wrapper is the single retry authority.
- **`complete(*, model, messages, max_tokens, system=None, temperature=None)`** → an
  `AICompletion` (`text`, `input_tokens`, `output_tokens`, `model`). Handles retry, timing,
  logging, and usage extraction; raises `AIClientError` (wrapping the SDK cause) on a
  non-retryable error or exhausted retries.
- **`_is_transient`** — classifies retryable errors via the SDK hierarchy.
- **Cost helper** (`app/ai/cost.py::estimate_cost`) — per-model pricing table → USD estimate.
- **Settings** — `ai_max_retries` (3), `ai_base_retry_delay_seconds` (1.0); model strings
  reuse the existing `anthropic_model_classification` / `anthropic_model_extraction`, now
  marked `TODO(models)` to verify. `.env.example` documents all of it.

## Retry Policy (transient-only)

| Error | Transient? | Behavior |
| --- | --- | --- |
| `RateLimitError` (429) | yes | retry with backoff + jitter |
| `InternalServerError` / 5xx (`APIStatusError`) | yes | retry |
| `APIConnectionError` / `APITimeoutError` | yes | retry |
| `BadRequestError` (400), `AuthenticationError` (401), `PermissionDeniedError` (403), `NotFoundError` (404), 422 | **no** | **fail fast** |
| non-SDK exceptions | no | fail fast |

Backoff is `base * 2**(attempt-1)` scaled by full jitter in `[0.5, 1.5)` to avoid a
thundering herd; capped at `ai_max_retries` attempts (default 3 → 2 sleeps then raise).

## Logging & Privacy (metadata only)

Every attempt/outcome logs **metadata only** — `model`, `input_tokens`, `output_tokens`,
`latency_ms`, `attempt`, outcome (`ai_call_succeeded` / `ai_call_failed`), and `error_type`
+ `transient` on failure. The **prompt and response content are never logged** — they carry
borrower PII. Content logging, if ever added, would be a redacted, debug-only option, not
the default. Two tests assert that captured logs (success and failure paths) exclude the
prompt, system prompt, and response content while including the metadata keys.

## Cost Estimation (an estimate to maintain)

`estimate_cost(model, input_tokens, output_tokens)` = `input*in_rate + output*out_rate` from
`PRICING` (USD per token). The table is **clearly marked an ESTIMATE** (`TODO(pricing)`) to
keep current with Anthropic's published pricing; an unknown model falls back to
`DEFAULT_RATE` (`0.0`) and logs `ai_cost_unknown_model`. Feeds `Extraction.cost_estimate`
(LP-16) and `Verification.total_cost_estimate` (LP-18) — callers persist it. **Not
billing-grade.**

## Decisions Made

- **ADR-117** — a single Anthropic client wrapper for all AI calls.
- **ADR-118** — retry transient errors only; log metadata, never content (PII).
- **ADR-119** — cost estimation via a maintained pricing table (estimate, not billing).

> **ADR numbering:** LP-36 used 114–116; the next free numbers were **117–119**.

## Assumptions

- **Classification (LP-38)** and **extraction (LP-39)** call this wrapper; **prompts** are
  defined there, not here (the wrapper is prompt-agnostic).
- **No streaming** in V1 (standard request/response).
- **Tests mock** the client — no real API calls, no real key, safe for CI.
- **Model strings & prices are configuration to verify** — the placeholders (reused from the
  LP-6 scaffolding) and the pricing table must be confirmed against current Anthropic docs
  before being relied upon; they are **not** presented as authoritative facts.
- The API key stays required for the app to start (CLAUDE.md), but a real AI *call* is what
  surfaces the missing-key error — so loading without a key is fine.

## Verification Performed

`uv run pytest` → **368 passed** (14 new), and `tests/ai/` passes **without a real API key**
(the SDK is mocked). ruff check + format, `mypy app/` (strict), pre-commit all clean.

- **Success** — `complete` returns the concatenated text + token usage; optional
  `system`/`temperature` are forwarded.
- **Retry-on-transient** — a 429 then a 5xx then success → 3 attempts, 2 backoff sleeps,
  returns the result.
- **Fail-fast** — a 400 raises `AIClientError` after **1** attempt (no retry); the cause is
  the `BadRequestError`.
- **Exhaustion** — persistent 5xx → 3 attempts then `AIClientError` (cause preserved).
- **Backoff** — with jitter fixed, recorded sleep delays strictly increase.
- **`_is_transient`** — 429/5xx/connection/timeout → transient; other 4xx and non-SDK
  errors → not.
- **Privacy** — success and failure logs contain the metadata keys and **none** of the
  prompt / system prompt / response content.
- **Cost** — known models compute `in*in_rate + out*out_rate`; an unknown model returns
  `0.0` and warns. Missing key raises `AIClientError` at call time.

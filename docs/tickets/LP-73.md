# LP-73 — Phase 2 Testing, Polish & Hardening (the Phase 2 capstone)

- **Ticket:** LP-73 — Phase 2 testing, polish & hardening
- **Epic:** Phase 2 — Document Handling (final ticket — closes the phase)
- **Status:** Completed
- **Date:** 2026-06-26

## Summary

Turns "the features are built + the unit tests pass" into "the **assembled system**
genuinely works." **No new features** — testing, hardening, polish. Driven by the
phase's lesson: **four bugs (a flush-timing bug, a Redis per-loop crash, a silent
AI-failure swallow, a host/worker storage split) all passed unit tests and broke on the
real stack — in the seams.** See **ADR-186** (operational robustness) and **ADR-187**
(real-stack testing + dev model).

## The six folded-in items (each a real gap this phase exposed)

1. **Worker by default** — removed the `profiles:` gate from the `worker` service;
   `docker compose up -d` now starts it. Async/AI features can't silently break. (ADR-186)
2. **Bounded retry + visible terminal-failed** — `app/tasks/retry.py` (`retry_or_terminal`,
   `MAX_RETRIES=3`, capped exponential backoff) wraps the needs + document tasks; on
   exhaustion it records a **visible** terminal-failed (`ai_needs_status=FAILED` /
   `status=FAILED`), never a silent permanent pending. Tested (`test_retry_policy.py`). (ADR-186)
3. **De-patched the LP-68 concurrency test** — it monkeypatched a per-loop Redis client
   (which *hid* the loop bug); it now runs against the **real loop-aware
   `get_redis_client`** (`test_needs_engine.py::_real_redis`). (ADR-187)
4. **Consistent local-dev worker model** — all-in-Docker with a shared storage volume;
   documented in `docs/development-workflow.md`. (ADR-187)
5. **Real-upload integration test** — Tier 1/2/3 through the **real storage backend
   (actual write then read) + real pipeline** (`test_phase2_real_stack.py`); a
   missing-file → graceful FAILED. The storage-split catcher. (ADR-187)
6. **S3 backend validation** — **honest scope:** the S3 backend is **not implemented**
   (`get_storage_backend` raises for `"s3"`, marked Phase 7), so there's nothing to
   validate against MinIO yet. Documented as deferred (Phase 7) — not overclaimed. (ADR-187)

## Real-stack integration tests (the phase's lesson)

`tests/integration/test_phase2_real_stack.py` exercises the **real** assembled pieces —
real storage (write **then** read), real DB, the real pipeline orchestration, the real
needs-satisfaction matching — **mocking only the AI model boundary** (classify / extract /
summarize / analyze):

- Tier 1 real upload → classify → extract → **satisfies the matching need**.
- Tier 2 → recognize + summarize; Tier 3 → generic analysis.
- A **missing stored file → graceful FAILED** (the storage-split regression).
- **MISMO import → floor + AI-reasoning** coexist (the differentiator's seam).

## Tenant-isolation pass

`tests/integration/test_phase2_tenant_isolation.py` sweeps **14 Phase-2 endpoints**
(documents list/detail/versions/download/delete/replace/resolve-staleness; needs
list/add/confirm/dismiss/waive/adjust; activity) and asserts a cross-company request
gets `404` — never another company's data, never a `403` that confirms existence.

  warnings, LP-46/47 loading/error/empty states) was reviewed; it was already finished —

- Seed: `app/scripts/seed_dev_data.py` already exercises the MISMO import + needs (it's
  Phase-2-aware); it runs and a fresh dev env demonstrates the tiers + the needs list.

the domain-tuning is ongoing.** We do not claim domain-completeness. (See the "Phase 2 —
Complete" section in `docs/phases/phase2.md` for the full deferred list.)

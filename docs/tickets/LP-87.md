# LP-87 — Additional Calculators (MI, Self-Employed, Reserves, Max Loan) + Overlay Admin UI

- **Ticket:** LP-87 — Arc B, two halves (four calculators + the overlay admin UI)
- **Epic:** Phase 3 — Verification Engine & Calculators
- **Status:** Completed
- **Date:** 2026-06-29

## Summary

Two halves, full-stack. **PART 1:** four additional calculators — mortgage insurance (MI/MIP), self-employed
income, reserves, and max loan — extending the LP-76/77 transparent / auto-populated / overrideable /
findings-coupled / deterministic pattern. **PART 2:** the overlay admin UI, closing the LP-80 hand-edited-JSON
deferral (view/edit lender overlays, admin-gated, reason-required, tenant-scoped, audited, effect-legible). See
**ADR-208** (calculators) + **ADR-209** (overlay admin).

## PART 1 — The Four Calculators (reuse LP-76/77 exactly)

Built as pure deterministic modules behind ONE shared `CalculatorView` shape + ONE shared
`calculator_overrides` table (calculator-discriminated) + ONE generic frontend component (`calculator-card.tsx`).

| Calculator | Consumes / feeds | Headline | Methodology |
|---|---|---|---|
| **MI/MIP** (program-aware) | reads LP-77 LTV; FHA consumes LP-84's `fha.mip.ufmip_rate` (175 bps) + the LTV-90% duration | monthly premium | PMI 80%/78% exact; rates starter |
| **Self-employed** | feeds the DTI income side | qualifying monthly income | Form 1084 (add-backs + 2-yr avg) — starter |
| **Reserves** | consumes LP-84's 60% retirement haircut + LP-76 PITI | months available | required months rule/DU-driven — starter |
| **Max loan** | inverts LP-76 DTI ceiling + LP-77 LTV limit + loan limit | max qualifying loan (binding wins) | FHFA/FHA loan limit — starter |

- **MI/MIP program-aware:** Conventional PMI (required > 80% LTV, cancels at 78% — exact); FHA MIP **consumes
  LP-84** (UFMIP 1.75% from the rule, annual rate starter 0.55%, the LTV-90% 11yr-vs-life duration).
- **Self-employed:** net profit + non-cash add-backs (depreciation/depletion/amortization/business-use-of-home),
  2-year average, declining-trend flag; grounded in Fannie Form 1084; the qualifying figure **feeds DTI**.
- **Reserves:** eligible = liquid + (vested retirement × factor) − down − closing (gifts/borrowed excluded);
  the **FHA 60% haircut** (LP-84) applies; months = eligible ÷ PITI (from DTI); available vs required.
- **Max loan:** inverts the DTI ceiling (→ max payment → invert amortization), the LTV limit (value × max-LTV),
  and the loan limit; the **binding (lowest)** constraint wins and is named.
- **Each reuses the LP-76/77 model:** transparent (the math shown as steps), auto-populated (from structured data
  + the sibling DTI/LTV calcs; Tier-2/manual where not extracted), overrideable inline (recompute flows,
  audited `CALCULATOR_OVERRIDDEN`), findings-coupled (the unresolved-findings alert), DETERMINISTIC (no AI).

### Methodology honesty

The MECHANISM is real + tested; the domain-judgment methodology (PMI rate, self-employed add-backs/averaging,
required reserves, loan limits) is `methodology.starter=True` (grounded in Form 1084 / FHA/FHFA limits, validate
with Priya). The deterministic arithmetic (MIP consumed from LP-84, the max-loan inversion) is solid.

## PART 2 — The Overlay Admin UI (closes the LP-80 deferral)

A thin UI + API OVER LP-80's existing `lenders.lender_overlays` JSON (not a new mechanism):

- **View** (`/admin/lenders` → `/admin/lenders/[id]`): the lender's overrides, each with its effect made legible
  (investor base → lender effective), + the audit trail.
- **Edit:** add/edit/remove override thresholds + each override's reason; a **required change reason**; save → the
  recomposed effect-legible view.
- **ADMIN-gated** (`require_role(UserRole.ADMIN)` on the router; the frontend role-gates for UX) ·
  **TENANT-scoped** (lender fetched within the company → 404 cross-company) · **reason REQUIRED** (422 otherwise) ·
  **AUDITED** (from→to via LP-80.5's `field_changes`/`audit_value`, stored in the overlay's own JSON `audit` list
  — no `activity_logs` schema change, since overlay edits are company-scoped, not loan-file-scoped) ·
  **EFFECT-LEGIBLE** (each override composed against the base rule by `rule_id`).
- **Seam (honest):** the live engine/calculators currently resolve the in-code STARTER_OVERLAYS; wiring the live
  engine to prefer a company's DB overlay (same file → different findings end-to-end) is a follow-on — the admin
  UI manages + makes legible the per-company overlay store, closing the hand-edited-JSON deferral.

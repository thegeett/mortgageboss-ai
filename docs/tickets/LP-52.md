# LP-52 — Stated-Financials Data Model + Migrations

- **Ticket:** LP-52 — stated-financials data model + migrations
- **Epic:** Phase 1.5 — MISMO Import
- **Status:** Completed
- **Date:** 2026-06-12

## Summary

Builds the database home for the **stated** financial data MISMO import produces — the *stated*
half of stated-vs-verified (the baseline documents verify against, and Phase-3 cross-checks). New
one-to-many models for stated income/employers/liabilities/assets (typed for Phase-3 comparison),
nullable extensions to Borrower/Property/LoanFile for the MISMO core fields they lacked, a MISMO
import record (catch-all + audit), and the Alembic migration. Backend only — models + migration +
tests; no API, no frontend, no mapping logic (LP-53 maps `ParsedMismo` into these).

## Existing-Schema Gap Analysis

What LoanFile/Borrower/Property already had vs. what MISMO needs:

| MISMO field | Status | Resolution |
|---|---|---|
| Borrower birth_date | exists | reuse `date_of_birth` |
| Borrower marital_status | exists (enum) | reuse |
| Borrower classification (Primary/Secondary) | exists | reuse `is_primary` (mapping sets it) |
| Borrower dependent_count / citizenship / declarations | **missing** | **add** (Integer / String / JSON) |
| Property estimated_value | exists | reuse |
| Property sales_contract_amount | exists | reuse `purchase_price` |
| Property usage_type | exists (enum) | reuse `occupancy_type` |
| Property valuation_amount / attachment_type / construction_method / financed_unit_count | **missing** | **add** |
| Loan base_loan_amount | exists | reuse `loan_amount` |
| Loan loan_purpose | exists (enum) | reuse |
| Loan mortgage_type | exists (enum) | reuse `loan_program` |
| Loan note_amount / note_rate_percent / lien_priority / amortization_type / amortization_months / application_received_date | **missing** | **add** |

All added columns are **nullable** — manual creation leaves them empty. Nothing was duplicated.

## What Was Built

**New one-to-many models** (`app/models/stated_financials.py`) — each Base + UUIDMixin +
TimestampMixin + SoftDeleteMixin, tenant-scoped transitively via the file, `ON DELETE CASCADE`:

| Model | FK | Fields |
|---|---|---|
| `StatedIncomeItem` | borrower | `monthly_amount` (Decimal), `income_type` (flexible str), `employment_income` (bool) |
| `StatedEmployer` | borrower | `employer_name`, `is_current` (bool) |
| `StatedLiability` | loan_file | `liability_type` (flexible str), `monthly_payment`, `unpaid_balance` (Decimals), `holder_name` |
| `StatedAsset` | loan_file | `asset_type` (flexible str), `value` (Decimal), `holder_name` |

**Import record** (`app/models/mismo_import.py`) — `MismoImport` (FK → loan_file): `source_format`,
`status` (`MismoImportStatus` COMPLETED/PARTIAL/FAILED — a CHECK-enum), `parse_warnings` (JSON),
**`catch_all`** (JSON — LP-51's everything-else), `raw_file_path` (reference to the preserved
original MISMO file for audit).

**Extensions** to Borrower (`dependent_count`, `citizenship`, `declarations` JSON), Property
(`valuation_amount`, `attachment_type`, `construction_method`, `financed_unit_count`), LoanFile
(`note_amount`, `note_rate_percent`, `lien_priority`, `amortization_type`, `amortization_months`,
`application_received_date`) — all nullable. Relationships added: Borrower → `stated_income_items` /
`stated_employers`; LoanFile → `stated_liabilities` / `stated_assets` / `mismo_imports` (all
`cascade="all, delete-orphan"`).

**Migration** `2680568f49c9` (autogenerated, then reviewed/cleaned): 5 new tables (correct types —
Numeric(14,2) money, Numeric(7,4) rate, JSON, Date, Boolean; FK `ON DELETE CASCADE`; FK indexes; the
`mismoimportstatus` VARCHAR+CHECK) + the nullable column additions. Header rewritten to repo style
(modern typing + the detect-secrets pragma). Applies (upgrade) and reverses (downgrade) cleanly.

## Shaped for Phase 3

The stated financials are typed for **deterministic comparison**: `Decimal` amounts (exact,
summable for DTI/reserves), typed/queryable categories, the monthly figures DTI needs. A **starter**
— refine with Priya / as Phase-3 rules firm up.

## One-to-Many Relationships

A borrower has many income items + employers; a file has many liabilities + assets (the real MISMO
file had 2 incomes, 3 employers, 9 assets, 10 liabilities). Modeled as rows with relationships +
cascade-from-parent, not columns.

## Tenant Scoping

No model carries its own `company_id`; all are scoped **transitively** via the loan file (ADR-053),
joining borrower→loan_file or directly loan_file, filtered by `company_id`. Each new model has a
per-model tenant-isolation test (file-level and borrower-level).

## Catch-All + Audit

LP-51's catch-all is stored as JSON on `MismoImport` (available later without re-parsing); the
original MISMO file is preserved via a `raw_file_path` storage reference (audit); the import event is
recorded (`MismoImport`: format / status / warnings / raw-file ref). PII in the catch-all/raw file is
tenant-scoped and never logged.

## Verification Performed

- **Round-trip:** each new model persists + reads back typed values exactly (Decimals, JSON, bool);
  the import record round-trips catch_all/parse_warnings/raw_file_path/status.
- **Relationships:** a borrower with 2 incomes + 2 employers; a file with 3 liabilities + 2 assets
  (via `selectinload`).
- **Cascade:** DB `ON DELETE CASCADE` — deleting a borrower removes its income items (file-owned
  liabilities survive); deleting the file removes its liabilities.
- **Tenant isolation:** file-level and borrower-level models — a query scoped (transitively via the
  file) to company A returns only A's rows, never B's.
- **Extended core fields** persist (note_rate `6.8750`, dependent_count, declarations, valuation,
  financed_unit_count) and default NULL on manual creation.
- **CHECK constraint:** an invalid `mismo_imports.status` is rejected (IntegrityError); flexible
  category strings (`MilitaryBasePay`, `HELOC`, …) are accepted.
- Migration **upgrade + downgrade** round-trip on the dev DB; `pytest tests/models/test_stated_financials.py`
  10 passed; full suite **606 passed**; ruff + mypy clean.

## Notes

- FK choices: income/employers → borrower; liabilities/assets → loan_file (deal-level for
  DTI/reserves). Documented in the model + ADR-160.
- MISMO category types are **flexible strings** (large/evolving); only small stable sets
  (MismoImportStatus) are CHECK-enums.
- Tests use `create_all` (ADR-039), so the new tables exist in the test DB automatically; the
  migration was verified separately on the dev DB.

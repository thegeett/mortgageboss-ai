## What Was Built

- **`app/(protected)/layout.tsx`** — client layout: `useRequireAuth()` (redirect to
  `/login` only after the silent-refresh check resolves; loader while `isInitializing`)
  + `<AppShell>` around `{children}`. The old `(app)` group was renamed/removed.
- **`components/layout/app-shell.tsx`** — `Sidebar` + `Header` over a scrollable
  `<main>` container (landmarks: aside/nav, header, main).
- **`components/layout/sidebar.tsx`** — wordmark + role-filtered nav, active-route
  indicator (`usePathname` + `isActivePath`), hidden below `md`.
- **`components/layout/header.tsx`** — mobile nav menu (below `md`), section title,
  user menu.
- **`components/layout/user-menu.tsx`** — shadcn dropdown: initials + name, **Log out**
  (LP-25 flow), disabled Profile/Settings placeholders.
- **`lib/navigation.ts`** — `NAV_ITEMS` (single source for desktop + mobile),
  `requiredRole` gating, `visibleNavItems(role)`, `isActivePath`.
- **Pages** — `dashboard/page.tsx` (moved; logout removed → header), `loan-files/page.tsx`
  (stub), `admin/page.tsx` (admin-gated nav; page reflects role).
- **`lib/navigation.test.ts`** — nav role-filtering + active-path tests.

## The Shell Pattern

- **Route group** — authenticated pages live in `app/(protected)/`; `/login` stays in
  `(auth)`, outside it. Groups don't affect URLs.
- **Structural protection** — the group layout guards once (not per page) and
  coordinates with LP-25's silent refresh: loader while `isInitializing`, redirect only
  once resolved unauthenticated → no flicker, no premature bounce on reload.
- **Adding a page** — drop `app/(protected)/<route>/page.tsx` (inherits protection +
  shell) and add a `NAV_ITEMS` line. See `docs/frontend-architecture.md`.

## Design Notes

- Built on the LP-5 tokens (primary blue `#2563EB`, cool grays, system font) and
  shadcn/ui (`dropdown-menu`, `button`, `badge`, `card`) — the home/login visual
  language, not a new one. Calm, uncluttered: white sidebar/header, gray-50 content,
  primary-blue active state.
- **Accessible**: `nav[aria-label="Main"]`, `header`, `main` landmarks; `aria-current`
  on the active link; focus-visible rings on links/triggers; keyboard-operable shadcn
  menus.
- **Responsive**: desktop-first; the sidebar collapses into a header dropdown below
  `md` (deep mobile polish deferred per scope).

## Notes

- **Frontend protection is UX, not security** (ADR-086) — a missing guard only affects
  chrome; the backend rejects unauthorized requests regardless.
- The `(app)` route group from LP-25 was consolidated into `(protected)`; the
  `useRequireAuth` hook remains the reusable utility, now invoked once in the layout.
- Playwright was used only as a throwaway verification tool and reverted (not added to
  `package.json`); the committed test surface stays the lightweight Vitest unit tests.

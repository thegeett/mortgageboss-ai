# LP-55 ‚Äî MISMO Upload Frontend (Primary Create-File Path)

- **Ticket:** LP-55 ‚Äî MISMO upload frontend
- **Epic:** Phase 1.5 ‚Äî MISMO Import
- **Status:** Completed
- **Date:** 2026-06-12

## Summary

The faucet on the MISMO pipeline (LP-51..54): the drag-and-drop "Upload MISMO" surface that makes
import visible and usable. It **reorients file creation** ‚Äî "Upload MISMO" is now the **primary**
path (the processor receives loan apps as MISMO from her loan officer), with "Create manually"
(Epic 4) as the secondary fallback. Upload ‚Üí the populated file opens immediately (import-directly),
showing the imported stated financials; partial imports show honest non-blocking warnings; errors
are friendly. Frontend + a minimal read endpoint (to expose the stated financials).

## What Was Built

- **Reoriented New File flow** (`app/(protected)/loan-files/new/page.tsx`): leads with the prominent
  `MismoUpload` drop zone; a subordinate "Create manually" reveals the reused Epic 4 `IntakeForm`
  (which can switch back to upload). MISMO is clearly primary (hierarchy, not a two-button row).
- **`MismoUpload`** (`components/intake/mismo-upload.tsx`): a react-dropzone hero (echoes LP-43)
  accepting XML/HTML; on success ‚Üí `router.push` to the created file + a success toast
  (import-directly); on error ‚Üí a friendly normalized message; "Importing‚Ä¶" spinner + disabled
  (double-submit prevention).
- **`useImportMismo`** + **`useStatedFinancials`** (`lib/api/mismo.ts`): the multipart upload
  mutation (calls LP-54) and the stated-financials read.
- **`StatedFinancialsSection`** (`components/file/overview/stated-financials-section.tsx`): the
  "Application data (stated)" card on the file Overview ‚Äî per-borrower income + employers, the
  file's liabilities and assets, the stated loan terms; the honest non-blocking warnings banner;
  loading skeleton + retry error state; hidden entirely for files with no stated data.
- **Minimal read endpoint** (backend, approved deviation): `GET
  /api/v1/loan-files/{id}/stated-financials` (`StatedFinancialsResponse`) ‚Äî read-only, tenant-scoped,
  exposes the already-stored stated financials + the latest import's warnings (needed to *display*
  them; no pipeline/model change). SSN masked.

## The Reoriented Flow (MISMO primary / manual secondary)

The processor's loan officer sends a MISMO file; she shouldn't re-type it. So the New File screen
leads with the MISMO drop zone, and manual entry is the fallback (reused, not rebuilt). The
information hierarchy makes MISMO the obvious primary action.

## The Payoff (import-directly)

On a successful upload there is **no preview/confirm** ‚Äî `router.push` opens the created, populated
file and a toast confirms ("Loan file imported ‚Äî the application data is filled in"). The opened
file shows the borrower/property/loan **and** the stated financials, already filled in.

## Stated-Financials Display (display-only)

"Application data (stated)" groups: **Income & employment** (per borrower ‚Äî each income item's
type + monthly amount, employers), **Liabilities** (type ¬∑ holder, monthly / balance), **Assets**
(type ¬∑ holder, value), and **Loan terms** (note rate, amortization, lien). SSN is masked
everywhere. **Display only** ‚Äî LP-56 adds editing.

## Honest, Non-Blocking Warnings

A partial import returns `warnings`; they persist on the import record and are read back with the
stated financials, so the opened file shows: "Imported ‚Äî a few fields need your attention: [‚Ä¶]. The
file was created ‚Äî you can fill these in." It reads as what-to-check, not a failure (sets up LP-56).

## Composed Patterns

Reused, not rebuilt: the **drag-drop** pattern (LP-43 dropzone), **error normalization** + friendly
messages (LP-46), **loading/spinner + double-submit** (LP-47), the **Epic 4 intake form**, and the
**file detail / Overview** view.

## Decisions Made

- **ADR-164** ‚Äî MISMO upload as the primary create-file path; import-directly; honest non-blocking
  warnings (+ the minimal stated-financials read endpoint).

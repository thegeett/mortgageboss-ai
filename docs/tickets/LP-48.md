# LP-48 — Seed Data Script

- **Ticket:** LP-48 — seed data script
- **Epic:** Epic 6 — Testing, Polish & Phase 1 Completion
- **Status:** Completed
- **Date:** 2026-06-12

## Summary

A dev-only, idempotent seed script that populates a fresh database with realistic demo data — so
the product can be demoed (LP-49) and reviewed with the domain expert (LP-50), and so development
against a populated DB is easy. It creates a company, an admin + a processor user, the UWM and
Sun-West lenders, and **three loan files in various workflow states**, each with a fake-PII
borrower, property, loan details, documents in various processing states with **pre-canned**
extractions (no AI calls), needs, and activity. Backend only (a script); no API change, no
features.

## What Was Built

`backend/app/scripts/seed_dev_data.py` — an async script:

- **Production guard** — `main()` refuses to run (exit 1, writes nothing) when
  `settings.is_production`.
- **Idempotent** — `_get_or_create_*` helpers check stable identifiers (company slug `summit`, user
  emails, lender slugs) and skip if present; loan-file seeding skips when the company already has
  files. Re-run = safe no-op (no duplicates, no unique-constraint crash).
- **`--reset`** — hard-clears the seeded company (DB cascade via `DELETE FROM loan_files…` →
  borrowers/properties/documents/extractions/needs/activity, then users, lenders, company) and its
  local storage subtree, then recreates fresh.
- **structlog** progress (metadata only — counts + created/skipped flags; never SSNs/secrets).
- Reuses the real models/services (`create_loan_file`, `create_extraction_version`, `log_activity`,
  `hash_password`, the storage backend) so records are correct and consistent with the schema.

## How to Run

```bash
cd backend
uv run python -m app.scripts.seed_dev_data          # check-and-skip (safe to re-run)
uv run python -m app.scripts.seed_dev_data --reset  # hard-clear seeded data + recreate
```

## No-AI Pre-Canned Extractions

COMPLETED (and the NEEDS_REVIEW) documents get their `Extraction` inserted **directly** — no AI, no
key, no broker. The `extracted_data` JSON is produced by constructing the **real** LP-39a models
(`PayStubExtraction`, `W2Extraction`, `BankStatementExtraction` with a `transactions` list) with
plausible fake values and calling `model_dump(mode="json")`, so the stored shape (typed core +
`{value, source:{page, snippet}}` + grouped `additional_sections`) always matches a live run and
can't drift. Values are realistic (employer, wages, dates, balances, 5 bank transactions) and fake.

## Idempotency

- **Check-and-skip** (default): company/users/lenders keyed on stable identifiers; loan files
  skipped if the company already has any. Verified: a second run created nothing (all
  `*_created=False`, `seed_loan_files_skipped existing=3`).
- **`--reset`**: clears the seeded company cascade + storage and recreates. Verified: clean
  recreate with a new company id.

## Safety

- **No real PII** — synthetic names/addresses; never-issued `900-xx-xxxx` SSNs written through the
  encrypted `EncryptedString` column and displayed masked (`***-**-3456`). No real data anywhere.
- **Production guard** — refuses + exits non-zero in production, writes nothing.
- **Dev-only credentials** — documented as local-dev conveniences.
- **Logs are metadata-only** — no SSNs/secrets logged.
- File bytes: a small **valid placeholder PDF** is generated and stored per document (so download
  works); it is not real document content. (No carried-over POC fixture PDFs were present.)

## Verification Performed

- **Runs + populates** (first run): created the company, 2 users, 2 lenders, 3 loan files,
  documents + extractions, needs, activity.
- **Re-run is idempotent**: all skipped, no duplicates, no crash.
- **`--reset`**: cleared (DB + storage) and recreated cleanly.
- **Production guard**: `ENVIRONMENT=production` → "Refusing to seed…", exit code 1, nothing
  written.
- **Spot-check (DB)**: both seeded users' passwords verify with `DevPassword123!`

  encrypted); a completed pay-stub extraction has the LP-39a shape (`employer_name = {value,
  source:{page, snippet}}`, version 1, status succeeded); the bank statement has 5 transactions +
  masked account `****6789`; a RECEIVED need is linked via `satisfied_by_document_id` with
  `satisfied_at` set.
- **Placeholder PDF**: valid structure (`%PDF-1.4` header, xref, trailer, `%%EOF`) — downloadable.
- `ruff check .`, `ruff format --check`, `mypy app/` all pass.

## Assumptions

- The schema may evolve → the script needs occasional updates (expected for a dev utility).
- File bytes are seeded as small placeholder PDFs (no carried-over POC fixtures were present); the
  records and download work, the content is not real.
- Extractions simulate the pipeline's output (inserted directly, no AI).
- The earlier minimal seed (`app.scripts.seed_dev`) is kept for a quick two-user setup.

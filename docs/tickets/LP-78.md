- **Ticket:** LP-78 — the AI cross-source layer + the APPLY→recompute loop
- **Epic:** Phase 3 — Verification Engine & Calculators (Arc B begins)
- **Status:** Completed
- **Date:** 2026-06-28

## Summary

The **"AI surfaces"** half of the locked two-layer principle arrives. LP-74 built the deterministic
*judge*; LP-78 builds the AI *perceiver* — **one general capability** that reads the borrower's stated
claims against what the documents prove and surfaces discrepancies (including ones no rule was written
for) as **structured findings** into LP-75's shared model. And it **closes the APPLY→recompute loop**:
an AI-surfaced finding, once a human applies it, changes the structured data → the calculators
recompute. Backend + a minimal trigger/staleness UI. See **ADR-192** and
[`phase-3-verification-design.md`](../phase-3-verification-design.md) §12.

## One General Capability — Not A Rule Per Check

The cross-source layer is a **single open-ended perception task**: "read the stated claims against the
document evidence and surface what doesn't line up." The prompt directs a general read-and-compare (not
a fixed checklist), guided toward high-value comparisons but **not limited** to them. Because the AI
reads and compares (rather than executing pre-written checks), it catches **known and novel**
discrepancies alike — the undisclosed obligation in a divorce decree that no rule covers. That is the
AI's superpower over deterministic rules: it finds what nobody enumerated.

## The Starter Set

Prompt guidance toward the high-value comparisons — stated income vs. income computed from pay
stubs/W-2s (>10% variance), stated employer vs. the document employer, stated gift vs. the gift
letter/deposit — demonstrated on the test file. The capability stays general (a test confirms an
unanticipated `undisclosed_obligation` surfaces alongside the starter comparisons). The full ~15-20 set
is LP-86.

## Structured Findings Only

The AI emits **typed** findings (type, amounts, source document + page + snippet, confidence, reasoning)
— never prose the deterministic layer interprets. They enter LP-75's **uniform** model with
`origin=ai_cross_source`, a confidence, and a source-location — identical in shape to deterministic
findings ("two generators, one findings model" completes). Parsing is defensive: a malformed AI
response yields no findings (nothing invented).

## AI Fallibility Acceptable

Findings are **for human review**, confidence-scored — the AI *surfaces candidates*, it does not decide.
They land **OPEN**, never auto-applied (a test asserts an obligation finding adds no liability until a
human applies it). A miss is backstopped by the processor; a false flag is overridden with a reason.

## The APPLY→Recompute Loop Closes

For recognized remediable types the emit attaches an **apply spec**; applying a finding (LP-75's hook)
changes the structured data → the DTI/LTV calculators (which read it live) recompute. End-to-end,
tested:

- **The classic case** — an undisclosed obligation → apply → added to liabilities → the DTI recomputes
  **higher**.
- **Income variance** → apply → the stated income corrected (lower) → the DTI recomputes **higher**.

LP-78 extends the apply hook with `correct_income` and makes applying mark verification stale (the
deterministic results are now out of date). This is the system's core interlock — AI perception → human
confirmation → deterministic recomputation.

## Manual Trigger + Staleness

The pass runs on a **manual trigger** (a "Run verification" button → the worker runs the AI call). Why
manual: cross-source compares two sides — meaningful only when both are assembled — and is a real AI
cost, so it runs deliberately. `verification_stale` is set on any document change (upload / type
override / replace) and when a finding is applied, and cleared when the pass re-runs — a visible
"re-run" indicator. **Auto-re-run is deferred** (the dial re-filters already-computed findings without
re-running — LP-79).

## Verification Performed

- **The general capability** — a discrepancy → a structured finding (origin=ai_cross_source, confidence,
  source-location) in LP-75's model; the prompt is general (asserted), not N hardcoded checks.
- **Starter comparisons + a novel discrepancy** — income variance / employer / gift surface, *and* an
  unanticipated `undisclosed_obligation` surfaces (capability general, not limited).
- **AI fallibility** — findings land OPEN, not auto-applied (the obligation adds no liability until applied).
- **The APPLY→recompute loop** — apply an undisclosed obligation → liabilities → DTI higher; apply an
  income variance → income corrected → DTI higher (end-to-end).
- **Manual trigger + staleness** — the pass clears staleness; a document change marks it stale; applying
  a finding marks it stale; no auto-re-run.
- **Structured-output parsing** — defensive (prose/fences tolerated; garbage → none; malformed items
  dropped; confidence clamped).
- **Endpoints** — POST triggers (RUNNING run + enqueue, patched); GET reports staleness + findings;
  cross-company 404.
- **PII** — assembled for the call; a `capture_logs` test asserts the borrower name + snippet appear in
  **no** log event.
- **No regression** — backend **1166 passed**; frontend **171 passed**; `ruff`/`mypy --strict` clean;
  `pnpm lint`/`typecheck`/`build` pass; the migration applies + reverses.

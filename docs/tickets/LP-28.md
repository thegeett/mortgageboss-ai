## What Was Built

### Schemas (`app/schemas/loan_file.py`)
`LoanFileCreate` (optional settable fields, no `company_id`), `LoanFileUpdate` (mutable
only; identifiers absent), `LoanFileSummary` (lean, derived `primary_borrower_name`, no
`inbox_token`), `LoanFileDetail` (summary + loan-officer + nested `BorrowerPublic`
[masked_ssn only] / `PropertyPublic`), `PaginatedLoanFiles`. `from_model` classmethods
build summary/detail from eager-loaded ORM objects.

### Service (`app/services/loan_files.py`)
A private `_scoped(company_id)` base (`scope_to_company` + `only_active`) underpins:
`list_loan_files` (status filter, `created_at desc`, offset/limit, full `total`, eager
borrowers), `get_loan_file` (UUID **or** display_id, eager borrowers+property),
`update_loan_file` (`exclude_unset`; never identifiers/ownership), `soft_delete_loan_file`
(`deleted_at = utcnow()`). All `flush`; the endpoint commits.

### Endpoints (`app/api/loan_files.py`)
POST/GET(list)/GET(one)/PATCH/DELETE under `/api/v1/loan-files`, each `CurrentUser`-gated
and scoped to `current_user.company_id`; `None` → `404`; writes `commit`. Wired into
`main.py`.

## Tenant Isolation (how it's enforced)

- **Company from the user, never the request** — create uses `current_user.company_id`;
  a `company_id` in the body is ignored (not in the schema, verified by test).
- **Scoped queries, not post-hoc checks** — `_scoped()` applies `scope_to_company` to
  every read, so another company's row never enters the result set.
- **`404`, not `403`** — out-of-company access finds nothing; existence isn't revealed.
- **Non-forgeable** — the scope derives from the validated token + live user.
- Verified: a Company A user cannot list / retrieve / update / delete Company B's files.

## Endpoint Contracts

| Method | Path | Auth | Scoping | Success | Errors |
| --- | --- | --- | --- | --- | --- |
| POST | `/api/v1/loan-files` | CurrentUser | company from user | `201` Detail | `401` |
| GET | `/api/v1/loan-files` | CurrentUser | scoped list | `200` Paginated | `401` |
| GET | `/api/v1/loan-files/{id\|display_id}` | CurrentUser | scoped get | `200` Detail | `401`, `404` |
| PATCH | `/api/v1/loan-files/{id\|display_id}` | CurrentUser | scoped get + update | `200` Detail | `401`, `404` |
| DELETE | `/api/v1/loan-files/{id\|display_id}` | CurrentUser | scoped get + soft delete | `204` | `401`, `404` |

## Decisions Made

- **ADR-093** — Tenant scoping by scoped queries; company from the authenticated user (404 not 403).
- **ADR-094** — Summary vs detail schemas; `inbox_token`/raw ssn never exposed.
- **ADR-095** — Address files by UUID or display_id; soft delete only.

> **ADR numbering:** LP-27 used 091–092; the next free numbers were **093–095**.

## Assumptions

- **Frontend is LP-29** — these are backend endpoints only.
- **Document/borrower/property management is later** — detail *reads* borrowers/property
  but doesn't manage them.
- **MISMO import is Phase 1.5; needs/verification/conditions are later.**
- **No role differences in V1** — any authenticated company user manages files.
- **Activity logging optional** — not added here (incremental, LP-30+).

## Verification Performed

- **Service** (`test_loan_files_crud.py`, 8): list/get scoping (A can't see B), dual
  identifier lookup, `exclude_unset` update + explicit-null clear, identifiers immutable,
  soft-delete exclusion, status filter + pagination, derived `primary_borrower_name`.
- **Endpoints** (`test_loan_files_endpoints.py`, 11): create returns DRAFT in the user's
  company (verified in DB), body `company_id` ignored; list scoped/filtered/paginated;
  retrieve by UUID and display_id; **cross-tenant retrieve/update/delete → 404**; PATCH
  mutable-only; soft delete → 404 after; detail masks ssn + hides `inbox_token` (raw ssn
  absent); all endpoints `401` without a token.
- **Full suite:** `uv run pytest` → **246 passed** (run 3× for stability). `ruff check`,
  `ruff format --check`, `mypy app/` (strict, 64 files) all pass.
- The endpoint tests use a **commit-safe** session (SAVEPOINT join mode) because these
  endpoints commit — the first place in the suite an endpoint commits inside a test.

## Notes

- **Pre-existing flaky test fixed (incidental).** The full-suite run surfaced a latent
  flake shared by four tamper tests (LP-14/22/23/24): they flipped the **last** base64
  char of a token/ciphertext, whose trailing bits can be unused — so ~1/16 of tokens
  decoded unchanged and the test failed (~23% chance per suite run across the four).
  Fixed all four to flip the **first** char (always significant). This touches prior
  tickets' test files but only to make the suite deterministically green.
- The list `status` query param shares a name with `fastapi.status`; it only shadows
  inside the handler body (which doesn't use the module), so it's harmless.

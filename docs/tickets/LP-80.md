# LP-80 — Starter lender overlays (UWM + Sun-West) + overlay enforcement

- **Ticket:** LP-80 — supply real overlay content + prove enforcement
- **Epic:** Phase 3 — Verification Engine & Calculators (Arc B)
- **Status:** Completed
- **Date:** 2026-06-29

## Summary

LP-74 built the overlay-application **mechanism** (override-by-id + add-custom + investor-default) with one
sample overlay. LP-80 supplies the real **content** — starter **UWM + Sun-West** overlays — and makes
enforcement demonstrable: the **SAME file produces DIFFERENT findings under UWM vs. Sun-West**. The overlays
are small diffs against the investor rule set, applied per-file via LP-74's composition (the file's target
lender selects the overlay). The DTI/LTV calculators' effective limit is now lender-specific. The specific
thresholds are **STARTER PLACEHOLDERS** for the domain expert (Priya) to validate — the mechanism is real;
the values are starter. See **ADR-196** and [`phase-3-verification-design.md`](../phase-3-verification-design.md) §14.

## What Was Built

- **Starter overlays** (`app/verification/overlays/starter.py`) — `UWM_OVERLAY` (slug `uwm`) and
  `SUNWEST_OVERLAY` (slug `sun-west`), each a small diff, clearly marked placeholder throughout.
- **The per-file lender binding** — the overlays are merged into `default_registry()` keyed by lender slug;
  the calculators + engine already resolve through `default_registry().resolve(program, lender_slug)` with the
  file's lender slug, so the binding works automatically (no rewiring).
- **A `reason` on `ThresholdOverride`** — each override records why the lender deviates (auditable + editable).
- **The seed** — Sun-West now supports Conventional too, so the same Conventional file can be assigned to UWM
  vs. Sun-West for the live demo.

## Overlays As Diffs

- **UWM** = one override (`conv.dti.back_end_max` → 45, tighter than the investor 50) + one custom rule
  (`uwm.reserves.min_months`, a reserves minimum the investor default lacks).
- **Sun-West** = one override (`conv.ltv.purchase_max` → 95) and **deliberately no DTI override**.
- Everything un-mentioned falls through to the investor default; the overlay value wins where specified. Each
  override carries a `reason`. Far smaller than the rule catalog — a diff, not a per-lender copy (LP-74's
  mechanism, LP-80's content).

## The Enforcement Proof (the headline)

The SAME file at 48% back-end DTI:
- **under UWM** → the DTI rule **fails** (48 > UWM's 45) → a finding fires.
- **under Sun-West** → the DTI rule **passes** (48 ≤ the investor 50) → no DTI flag.

Same data, different lender, different findings. Proven at the engine layer (pure facts) **and** the
calculator layer (`limit.status` over vs pass). The demo moment: "fine for Sun-West, flags for UWM, because
UWM is stricter on DTI."

## The Per-File Lender Binding

The file's `lender_id` → the lender's `slug` → the overlay (`default_registry()` keyed by slug). A file with
no lender gets the investor defaults. Lenders are per-company rows; the starter overlay content is universal
config keyed by slug (the right representation for shared placeholders) — per-company hand-edited overlays are
LP-87.

## The Effective-Limit Connection

The DTI/LTV calculators (LP-76/77) already resolve the effective limit via LP-74; LP-80's content makes it
lender-specific:
- A **UWM** Conventional file → DTI limit **45** (`source=overlay`, `lender_slug=uwm`).
- A **Sun-West** file → DTI limit **50** (`source=program_default`) and purchase-LTV cap **95** (`source=overlay`).
- Retarget the same file's lender → the limit changes. No calculator code changed.

## Honest Starter Scoping

The UWM / Sun-West thresholds are **STARTER PLACEHOLDERS** — NOT authoritative lender requirements (that
knowledge isn't available yet). They are a small, plausible set for Priya to validate and correct, marked
`STARTER PLACEHOLDER` in the module docstring, every override `reason`, every custom-rule description/citation,
and the docs. The **mechanism is real; the values are starter**. Structured so replacing/extending is trivial.

## Verification Performed

- **Overlays load + are diffs** — UWM + Sun-West parse; each a handful, far smaller than the rule catalog.
- **Override-by-id** — UWM's `conv.dti.back_end_max` resolves to 45 (`overlay_applied=uwm`), identity/logic preserved.
- **Add-custom** — UWM's `uwm.reserves.min_months` is added.
- **Investor-default fall-through** — Sun-West does not override DTI → 50 (`overlay_applied=None`); it overrides LTV (95).
- **THE ENFORCEMENT PROOF** — the same 48% file fails the DTI rule under UWM, passes under Sun-West (engine + calculator).
- **The effective-limit connection** — the DTI/LTV calculators reflect the overlays (UWM 45 / Sun-West 95, with source).
- **The per-file lender binding** — the file's lender selects the overlay; no lender → defaults; per-company lenders.
- **Reasons marked placeholder** — every starter override carries a reason marked a starter placeholder.
- **No regression** — backend **1220 passed**; `ruff`/`mypy --strict` clean (no frontend touched).

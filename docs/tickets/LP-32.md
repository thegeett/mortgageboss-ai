## The Option-A Orchestration (ADR-105)

`submitIntake` runs sequentially, **file-first**:
1. `POST /loan-files` — the **gate**. On failure: show a form-level error, **stay** (retryable).
2. `POST /loan-files/{id}/borrowers` (primary, `is_primary=true`) — best-effort.
3. `POST /loan-files/{id}/property` (only if any property field entered) — best-effort.

If the file is created but a best-effort step fails, it's collected as a `warning`; the form
**navigates to the file anyway** and shows a non-blocking `toast.warning`. **No rollback.**
On full success → `toast.success` + redirect to `/loan-files/{display_id}`; the
`["loan-files"]` query is invalidated so the dashboard reflects the new file.

## Validation Choice (ADR-106)

**Light / DRAFT-friendly**: the only required fields are the primary borrower's **first +
last name**; everything else is optional and **format-checked only when entered** (email,
9-digit SSN, 2-letter state, ZIP, non-negative amounts). Empty = "not provided".

## SSN Handling

Input-only: sent once to the borrower endpoint (encrypted at rest), returned only as
`masked_ssn`, **never console-logged** (no logging of the form payload), not persisted
client-side beyond submit. Verified below.

## Scope

V1 captures the **primary borrower only** (co-borrowers deferred — the API supports
multiple; addable later). Native styled selects instead of a JS dropdown (no new dependency;
better form/mobile UX). Lender dropdown shows "No lenders configured" until lenders are
seeded (LP-48).

## Notes

- The native-`<select>` wrapper is a deliberate choice over a Radix/shadcn Select: fully
  accessible, keyboard/mobile-friendly, integrates with react-hook-form via a forwarded ref,
  and adds no dependency for simple option lists.
- Playwright was a throwaway verification tool (already present from earlier tickets, not
  re-added to `package.json`).
- The form-created `LF-CZWC` (and the LP-31 demo files) remain in the dev DB for your manual
  testing.

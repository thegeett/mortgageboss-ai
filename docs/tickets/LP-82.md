## The Researched Grounding (current Selling Guide, June 2026)

The research corrected folk-knowledge — these are the starter values used: **4-month** document age on the
note date (B1-1-03), not 30 days; the **most recent W-2 + pay stub** for base income (the B3-3 rewrite of
03/2026), not two years of W-2s; a **2-year** self-employment history (B3-3.5-01); gift/asset/retirement
verification (B3-4.x). The **large-deposit threshold** and **reserves** are **DU-message-driven / program-
driven** and are marked STARTER placeholders, NOT Selling-Guide constants.

## The Starter / Validate-with-Priya Posture

Every rule carries `starter=True` + a `notes` caveat; the module header + this doc say so. The mechanism is
real and tested; the **content is grounded-but-starter** pending the expert's validation — because the
Selling Guide changes frequently (B3-3 was rewritten 03/2026), many requirements are DU-automated, and lender
overlays (LP-80) tighten the investor defaults. Citations are **researched, not invented** — where a
subsection is uncertain the source carries `to_verify=True`.

## Typed-Core Promotions

`build_file_facts` derives three new facts from the stated structured data (`assets.gift.total_amount`,
`assets.retirement.total_amount`, `income.self_employment.monthly_amount`), set only when present so the
"documentation required" rules fire on a real file and are simply not-evaluated otherwise. The remaining
rules read a canonical typed-field path whose fact isn't produced yet — their `notes` say "typed-core
promotion pending: <fact>" and the engine records them not-evaluated (graceful) until the fact lands.

  deposit then fails) and `conv.income.self_employment_history` (24 → 12) by `rule_id`.
- **Test-file evaluation** — a Mahesh-shaped file (self-employment income + $56k gift + $243k retirement) →
  the promoted facts appear and the self-employment / gift / retirement rules fire.
- **No regression** — backend **1249 passed** (two engine tests updated to count by status, since real rules
  now fire); `ruff`/`mypy --strict` clean. No migration (rules are code config).

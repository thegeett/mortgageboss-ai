# LP-14 — Borrower and Property Models

- **Ticket:** LP-14 — Borrower and Property models with application-level SSN encryption
- **Epic:** Epic 2 — Database & Models (sixth ticket)
- **Status:** Completed
- **Date:** 2026-06-10

## Summary

Created the **`Borrower`** and **`Property`** models — the people and real estate
attached to a loan file. Two new patterns land here:

1. **A one-to-many off the loan file** — a file has one or more borrowers (a
   primary plus co-borrowers), and exactly one subject property (a one-to-one
   enforced by a unique constraint).
2. **Application-level column encryption** for the borrower **SSN**, the most
   sensitive field in the system (GLBA-covered). A custom `EncryptedString`
   SQLAlchemy type encrypts on write and decrypts on read using Fernet; the
   database stores only ciphertext and the key lives in settings, never in the
   database. This **reconsiders the LP-9 assumption** that pgcrypto would encrypt
   this field (**ADR-051**).

Both models are company-scoped **transitively** through their loan file — neither
carries its own `company_id` (**ADR-052**).

Implemented in **three reviewed stages**: (1) encryption service + `EncryptedString`
type + tests in isolation (the security core first), (2) `Borrower` + `Property`
models + enums + migration, (3) model tests (encryption-at-rest, masking,
no-leak, multi-borrower, tenant isolation, one-property-per-file) + docs.

## What Was Built

### Encryption service (`app/core/encryption.py`)

`get_cipher()` returns a cached `Fernet` built from `settings.encryption_key`.
`encrypt_value(str|None) -> str|None` and `decrypt_value(str|None) -> str|None`
do the value-level crypto; `None`/empty map to `None` (the column stays SQL
`NULL`). Decrypt failures raise `ValueError` **without echoing the token**.

### Custom type (`app/models/encrypted_types.py`)

`EncryptedString` is a `TypeDecorator` over `Text`: `process_bind_param` encrypts
on write, `process_result_value` decrypts on read. Declared like any column;
encryption is invisible to model code.

### Config

`ENCRYPTION_KEY` is a **required** setting (`min_length=44`, no default) — the
app refuses to start without it, like `JWT_SECRET_KEY`. Added to `.env`
(generated Fernet key, gitignored), `.env.example` (placeholder + generation
hint), and the Backend CI workflow (throwaway CI key).

### Models (`app/models/borrower.py`, `app/models/property.py`)

- `Borrower` — `loan_file_id` FK (`ondelete=CASCADE`, indexed); required
  `first_name`/`last_name`; nullable `middle_name`, `ssn` (`EncryptedString`),
  `date_of_birth`, `email`, `phone`, `marital_status`; `is_primary` (bool),
  `borrower_position` (int). `full_name`, `masked_ssn` (`***-**-1234`), and a
  PII-safe `__repr__` (position + `loan_file_id` only).
- `Property` — `loan_file_id` FK (`ondelete=CASCADE`, **unique** → one-to-one);
  nullable address fields, `property_type`/`occupancy_type` enums,
  `estimated_value`/`purchase_price` as `Money`.
- `LoanFile` gains `borrowers` (one-to-many, `order_by` position,
  `cascade="all, delete-orphan"`) and `property` (`uselist=False`).
- Enums: `MaritalStatus`, `PropertyType`, `OccupancyType` (all VARCHAR + CHECK).

### Migration (`82c2b1989cce_create_borrowers_and_properties.py`)

Creates both tables. Hand-cleaned to match the repo's migration style; the `ssn`
column is rendered as `sa.Text()` (not the custom type) to keep the migration
**self-contained and decoupled from app code** — the DB column is just text
holding ciphertext.

## Security: SSN encryption (ADR-051)

- **Application-level**, not pgcrypto — the key never reaches the database, so a
  database-only compromise yields **ciphertext only**.
- **Fernet** = authenticated AES-128-CBC + HMAC; tampering is detected on decrypt.
- **Non-deterministic** (fresh IV per write): the same SSN encrypts differently
  each time, so an encrypted column **can't** be queried by equality, indexed, or
  made unique. Fine for SSN (never queried by it).
- **Never leaks**: no SSN in repr, logs, or errors; `masked_ssn` for display.
- **Verified at rest**: a test reads the raw `ssn` column with SQL (bypassing the
  type) and asserts it is ciphertext, never the plaintext or bare digits.
- **Scope (V1):** only the SSN is encrypted. Date of birth is unencrypted (lower
  risk, needed for matching). Key rotation and secret-manager integration are
  Phase 7; a single active key from settings is used.

## Decisions Made

- **ADR-051** — Application-level encryption for the SSN, not pgcrypto
  (reconsiders LP-9). pgcrypto stays enabled but unused.
- **ADR-052** — Borrowers and properties are company-scoped transitively through
  the loan file (no own `company_id`); queries join through the file and scope it.

## Assumptions

- **Names required, the rest nullable** — a borrower is minimally a named person;
  SSN/DOB/contact/marital status arrive incrementally (manual entry, MISMO,
  extraction).
- **Address/type/occupancy nullable on Property** — the subject property is often
  "TBD" early in a purchase.
- **One property per file in V1** — enforced by the unique constraint; multi-
  property files (blanket loans) are out of scope.
- **`is_primary` not DB-enforced** — exactly-one-primary is set by the creating
  service in V1; a partial unique index could enforce it later.

# LP-29 — Borrower and Property API Endpoints

- **Ticket:** LP-29 — Borrower and property API endpoints (nested, tenant-scoped, SSN masked)
- **Epic:** Epic 4 — Loan File CRUD (second ticket)
- **Status:** Completed
- **Date:** 2026-06-11

## Summary

Nested CRUD for the owned children of a loan file: **borrowers** (a collection) and the
subject **property** (a per-file singleton). Both are scoped **transitively through the
parent file** (no own `company_id`), and the borrower SSN is handled **in-but-masked-out**
(accepted as input, stored encrypted, returned masked only).

## What Was Built

### Schemas
`borrower.py` — `BorrowerCreate`/`BorrowerUpdate` (accept raw `ssn`) and
`BorrowerResponse` (**`masked_ssn` only**, no `ssn` field). `property.py` —
`PropertyCreate`/`PropertyUpdate`/`PropertyResponse`.

### Services
`borrowers.py` — `list_borrowers` (active, by position), `get_borrower` (matches
`borrower_id` **and** `loan_file_id`), `create_borrower` (SSN → `EncryptedString`;
first-borrower primary/position defaults; demote-others when primary),
`update_borrower` (`exclude_unset`; re-encrypts a provided `ssn`; demote on
`is_primary=True`), `soft_delete_borrower`. `properties.py` — `get_property`,
`create_property` (raises `PropertyExistsError` if one exists → 409), `update_property`,
`soft_delete_property`.

### Endpoints
A shared **`ScopedLoanFile`** dependency (`get_scoped_loan_file` in
`app/api/dependencies.py`) resolves the path file scoped to `current_user.company_id`
and 404s if absent — declared by every borrower/property route, so the file gate runs
first. `api/borrowers.py` (list/add/get/update/delete) and `api/property.py`
(get/create/update/delete) under `/api/v1/loan-files/{file_identifier}/…`; writes commit.

## Transitive Tenant Scoping (the file gate)

Children have no `company_id`; the parent file is the scope. Every route declares
`ScopedLoanFile`, which calls `get_loan_file(company_id=current_user.company_id,
identifier=file_identifier)` **first** — a file in another company (or absent) → `404`,
and the borrower/property is never reached. `get_borrower` also matches `loan_file_id`,
so a borrower id from a different file is `404` under this file. Verified: a Company A
user cannot list/add/get/update/delete borrowers or property on a Company B file.

## SSN Handling (in-but-masked-out)

- **In:** create/update accept a raw `ssn`, written to the `EncryptedString` column →
  **encrypted at rest** (Fernet, ADR-051). Verified by reading the raw column with SQL
  (ciphertext, not the SSN).
- **Out:** `BorrowerResponse` has **no `ssn` field** — only `masked_ssn` (`***-**-1234`).
  Tests assert no response (create/list/get/update) contains the raw SSN or its digits.
- **Logs:** request bodies aren't logged; the model `__repr__` already omits the SSN.

## Property Singleton Semantics

One active property per file (DB `unique(loan_file_id)`). `GET`/`PATCH`/`DELETE` act on
the single property (`404` when none); a second `POST` → **`409`** (`PropertyExistsError`).

## Decisions Made

- **ADR-096** — Nested endpoints; transitive tenant scoping via the file.
- **ADR-097** — SSN in-but-masked-out at the API boundary.
- **ADR-098** — Property per-file singleton (409); minimal primary-borrower logic.

> **ADR numbering:** LP-28 used 093–095; the next free numbers were **096–098**.

## Assumptions

- **Frontend is later** — backend only.
- **MISMO import is Phase 1.5; `stated_*` application data is Phase 1.5** — borrowers/
  property are entered manually here.
- **Minimal primary/position logic** — first borrower primary/position 1; demote-on-
  primary; otherwise client-managed.
- **No role differences beyond authentication.**
- **Re-create property after soft-delete is out of scope** — DB unique constraint holds
  for the soft-deleted row (a partial unique index would be a later model change).

## Notes

- Borrower/property carry no `company_id` (ADR-052/053); the file gate is the entire
  tenant boundary for them — hence the shared `ScopedLoanFile` dependency.
- The raw SSN is structurally unserializable on the way out: `BorrowerResponse` simply has
  no `ssn` field, so masking can't be bypassed by a serialization mistake.

## The Two Backend Reads (scoped, LP-29 pattern)

- `services/needs_items.py::list_needs_items` + `api/needs.py` — `GET
  /loan-files/{file_identifier}/needs`, declares `ScopedLoanFile` (file gate first),
  active items ordered **blocking-first** then oldest. `schemas/needs_item.py::NeedsItemPublic`.
- `services/activity_log.py::list_recent_activity` + `api/activity.py` — `GET
  /loan-files/{file_identifier}/activity`, file gate first, **most-recent-first**, capped at
  20. `schemas/activity.py::ActivityPublic`.

Both reuse the LP-29 gate, so a Company A user gets `404` for a Company B file (tested). No
`company_id` from the request. The borrower card reuses the existing LP-29 `/borrowers`
read (richer fields: marital_status/email/phone) — no detail-schema change needed.

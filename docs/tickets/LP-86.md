## Summary

non-deterministic) into DETERMINISTIC rules ‚Äî a new `xsrc.*` category in the verification engine that runs every

discovery and DE-DUPLICATED (it defers on a canonical type a deterministic rule fired this run). This is the

## The Graduation (the heart of LP-86)

The AI cross-source layer catches novel discrepancies but flickers on the known ones ‚Äî the diagnostic signal was
the **driver's-license-address-equals-subject-property** finding appearing on one run, not the next, because it
is actually deterministic logic (compare two addresses), not perception. LP-86 makes the reliable, enumerable
checks **rules**: the DL finding now fires **every run, identically**, as `xsrc.address.dl_equals_subject`.

## The Distinct Category

A cross-source rule reads MULTIPLE fields ACROSS sources (not one threshold vs one field like LP-82..85). The
pure engine is `app/verification/cross_source/`:

- **`facts.py`** ‚Äî `CrossSourceFacts` (values tagged by source + the diff lists) + `SourcedValue` / `ObligationRef`.
- **`rules.py`** ‚Äî `CrossSourceRule` (stable `rule_id`, the owned `canonical_type`, category, severity, TEMPLATED
  `template`, a pure `check`, threshold-as-data, `program=None` agnostic, `starter=True`) + the 18 rules + the
  pure checks + `OWNED_CANONICAL_TYPES` + `apply_cross_source_overlay`.
- **`engine.py`** ‚Äî `evaluate_cross_source(facts, program, rules)` ‚Äî pure: same facts ‚Üí same findings, in order.

## The Rules (~18)

| Area | Rules (`xsrc.*`) | Owned canonical type ¬∑ severity |
|---|---|---|
| Identity | `identity.name_consistency`, `identity.ssn_consistency`, `identity.dob_consistency`, `address.current_address_consistency` | identity_discrepancy ¬∑ YELLOW (**SSN RED**) |
| Address red-flag | `address.dl_equals_subject` (**the graduate**), `address.employer_equals_subject` | property_address_discrepancy ¬∑ YELLOW |
| Income | `income.stated_vs_documented` (variance %, **threshold-as-data**), `income.employer_name_consistency`, `income.employer_count_matches_items` | income_variance / employer_mismatch ¬∑ YELLOW |
| Liability | `liability.undisclosed_debt` (**APPLY‚Üírecompute**), `liability.stated_not_on_report` | liability_discrepancy ¬∑ YELLOW |
| Asset | `asset.stated_missing_document` (kept), `asset.large_deposit_unsourced`, `asset.gift_without_letter` | missing_documentation / asset_discrepancy / gift_discrepancy ¬∑ YELLOW |
| Terms/property | `terms.price_vs_contract`, `terms.loan_vs_documented`, `property.subject_address_consistency`, `property.occupancy_vs_evidence` | terms_discrepancy / property_address_discrepancy / occupancy_discrepancy ¬∑ YELLOW |

Two new canonical types (`terms_discrepancy`, `occupancy_discrepancy`) join the taxonomy so the deterministic
rules can own them (and the AI can defer). Output is `origin=deterministic_rule`, `confidence=1.0`.

## De-Duplication (the graduation mechanics)

`run_cross_source` now runs the deterministic pass FIRST (via `cross_source_deterministic.run_cross_source_deterministic`),
which returns the set of canonical types it FIRED. The AI emit loop then DROPS any raw finding whose `type` is in
that set (`ai_deferred` in the log). This is **run-scoped**: if the deterministic pass was silent on a type
(e.g. its Tier-2 facts aren't loaded), the AI still surfaces it ‚Äî no coverage loss ‚Äî and the AI always keeps the
novel **"other"** bucket + `co_borrower_discrepancy`. No double-reporting of a fired discrepancy; the stable,
templated deterministic finding is the one shown.

## The Consistency Payoff (option D)

The promoted checks now run every time, identically ‚Äî no AI, no recall variance, no flicker. This completes the
arc: LP-78.1 (caching) + LP-81 (stable identity / merge / templated wording) made the AI findings as consistent
as possible; LP-86 makes the reliable ones FULLY deterministic. Verified: the DL finding is byte-identical
across 3 re-runs (the prior open one superseded + re-emitted).

## Cross-Links (wired, not rebuilt)

- **Asset/gift/deposit ‚Üî LP-82:** the cross-source checks compare ACROSS sources; LP-82's single-source rules
  check one field. The cross-source rules surface DATA discrepancies, never computed DTI/LTV (the calculators').

## The Absent-Data Guard (honesty)

A set-difference check (`stated_not_on_report`, `undisclosed_debt`) only fires when the credit report is
actually present ‚Äî an empty side is "not loaded", not a discrepancy (this guard fixed a false-positive cascade
on every file with stated liabilities). Many facts (credit-report liabilities, contract price, documented income,
occupancy evidence) are Tier-2 / promotion-pending; their checks produce nothing until the fact lands (graceful,
as LP-83..85). `build_cross_source_facts` populates what's readily available today ‚Äî names across sources, the
DL address vs the subject property, stated income, employers + the income-item count, stated liabilities, the
gift-letter presence.

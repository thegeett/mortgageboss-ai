# LP-53 — MISMO → Models Mapping + File Creation Service

- **Ticket:** LP-53 — MISMO → models mapping + file-creation service
- **Epic:** Phase 1.5 — MISMO Import
- **Status:** Completed
- **Date:** 2026-06-12

## Summary

The bridge between the MISMO parser (LP-51) and the stated-financials models (LP-52):
`create_loan_file_from_mismo` takes a `ParsedMismo` and creates a fully-populated `LoanFile` —
borrower(s), property, loan terms, stated income/employers/liabilities/assets, the catch-all, a
stored raw MISMO file (audit), and a `MismoImport` record. Parsed data in → a populated, usable
loan file out. Backend only — the service + tests; no endpoint (LP-54), no frontend (LP-55).

## What Was Built

`app/mismo/import_service.py` — `create_loan_file_from_mismo(db, *, parsed, company_id, raw_content,
source_format=None, actor_user_id=None) -> LoanFile` (async) + `MismoImportError`.

## The Mapping (ParsedMismo → models)

| Parsed | → | Model |
|---|---|---|
| `loan` (base_loan_amount, note_amount, note_rate_percent, mortgage_type→program, loan_purpose, lien_priority, amortization_type/months, application_received_date) | → | `LoanFile` (via Epic 4 `create_loan_file` + set the MISMO fields) |
| `property` (address, estimated_value, sales_contract→purchase_price, usage_type→occupancy, valuation_amount, attachment_type, construction_method, financed_unit_count) | → | `Property` (via Epic 4 `create_property` + set the MISMO-only fields) |
| `borrowers[]` (name, dob, marital, dependents, classification→is_primary, citizenship, email/phone, declarations, **SSN**) | → | `Borrower` (constructed directly; SSN via the encrypted column) |
| borrower `income_items[]` | → | `StatedIncomeItem` (per borrower) |
| borrower `employers[]` | → | `StatedEmployer` (per borrower) |
| `liabilities[]` | → | `StatedLiability` (per file) |
| `assets[]` | → | `StatedAsset` (per file) |
| `catch_all` | → | `MismoImport.catch_all` (JSON) |
| `parse_warnings`, `source_format`, raw file | → | `MismoImport` (+ `raw_file_path` to the stored original) |

MISMO category strings → our small domain enums (marital/program/purpose/occupancy) with **unknown →
None**; large/evolving categories (income/liability/asset types) stay flexible strings (LP-52).
Money/rates are mapped as the parsed **`Decimal`s** (no float drift); dates as dates.

### Convergence with manual creation

The `LoanFile` is created via Epic 4's **`create_loan_file`** and the `Property` via
**`create_property`**, so a MISMO file is the *same* model (generated `display_id`/`inbox_token`,
`DRAFT` status, same relationships, same downstream) as a manually-created one — they converge, no
parallel creation. Borrowers are constructed directly (not via `create_borrower`) only so they can
carry the MISMO-only fields, tolerate a non-`EmailStr` email, and position multiple borrowers; the
resulting `Borrower` is identical.

## Transactional

The service **flushes**; the caller (the LP-54 endpoint) **commits** — the codebase convention — so
the whole creation is one **all-or-nothing** transaction. A mid-creation failure (verified by
forcing the raw-file store to raise inside a savepoint) rolls the entire creation back: **no
half-created file**.

## Partial-Parse Handling (import-directly)

- Missing optional fields → the file is **still created** with what's present (missing → `None`);
  `parse_warnings` are stored on the `MismoImport` (status `PARTIAL`) and surfaced later (LP-56). Not
  blocked.
- **Floor:** if there is *no* borrower **and** no loan at all → `MismoImportError` (the endpoint
  surfaces it). Anything above the floor (a borrower **or** loan present) creates the file (verified:
  loan-but-no-borrower still creates).

## PII

- The SSN is stored **only** through the existing encrypted `Borrower.ssn` column (Fernet) and is
  **never logged** (verified: the raw DB column is ciphertext, not the plaintext SSN).
- Logging is **metadata-only** — `loan_file_id`, counts (borrowers/liabilities/assets),
  `source_format`, warning count — never names/SSN/amounts/raw content (verified: captured logs
  contain none of the SSN, names, or amounts).
- The raw MISMO file is stored access-controlled (tenant-scoped path under the company/file) and
  never logged.

## Decisions Made

- **ADR-162** — MISMO import service: the mapping seam; converges with manual; transactional;
  partial-parse create+warn.

## Assumptions

- LP-54 (upload endpoint) calls this service and owns the commit.
- Phase-3 verification consumes the stated data this populates.
- Re-import / versioning is deferred (the import record sets it up).
- FK choices per LP-52 (income/employers → borrower; liabilities/assets → file).
- SSN stays on the existing encrypted Borrower path; PII is never logged.

## Notes

- The mapping is the **single** MISMO↔model seam — the parser and the models stay ignorant of each
  other.
- Known gap: the MISMO borrower address has no typed `Borrower` column (only the subject property has
  an address); parsed but not persisted to a typed field — a later model change.

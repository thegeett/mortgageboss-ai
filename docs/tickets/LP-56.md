# LP-56 — Edit Imported Data (Reviewable / Editable After Import)

- **Ticket:** LP-56 — edit imported data
- **Epic:** Phase 1.5 — MISMO Import
- **Status:** Completed
- **Date:** 2026-06-13

## Summary

Completes the import-directly safety net (LP-54/55): the data MISMO import populated is now
**reviewable and editable**, so a parse gap or wrong value — the very things the LP-55 parse warnings
flag — is corrected on the opened file instead of by re-importing or re-typing. The original MISMO
(raw file + `MismoImport` record) is preserved; editing changes only the *derived* application data.
Edits are audited, tenant-scoped, validated, and SSN-safe. Scope is **correct the import + add/remove
rows**, not a from-scratch application builder.

## Reuse vs. Add (the core decision)

Editing splits cleanly along the existing seam (see ADR-165):

- **Core fields → REUSE Epic-4 PATCH.** The borrower/property/loan-file PATCH endpoints already apply
  fields generically (`model_dump(exclude_unset=True)` → `setattr`), so making the MISMO-specific
  core fields editable needed only **extending the Update schemas** — no new endpoints, no new UI.
  SSN reuses the existing `BorrowerUpdate.ssn` encrypted re-enter path.
- **Stated financials → ADD multi-row CRUD.** Genuinely new: the four LP-52 stated kinds are
  one-to-many, so they need create / update / delete-row endpoints.

## What Was Built

**Backend**

- **Extended Update schemas** (makes the MISMO core fields editable through the *existing* PATCH
  endpoints):
  - `BorrowerUpdate`: `dependent_count`, `citizenship`, `declarations`.
  - `PropertyUpdate`: `valuation_amount`, `attachment_type`, `construction_method`,
    `financed_unit_count`.
  - `LoanFileUpdate`: `note_amount`, `note_rate_percent`, `lien_priority`, `amortization_type`,
    `amortization_months`, `application_received_date`.
- **Stated-financials CRUD** (`app/api/stated_financials.py`, new router mounted at `API_V1_PREFIX`):
  12 endpoints — POST under the file (`/loan-files/{id}/stated-liabilities`, `…/stated-assets`) and
  under the borrower (`/loan-files/{id}/borrowers/{bid}/stated-income`, `…/stated-employers`), plus
  PATCH/DELETE by row id (`/stated-liabilities/{id}`, `/stated-assets/{id}`,
  `/stated-income-items/{id}`, `/stated-employers/{id}`). Add builds `Model(**model_dump())`; update
  setattrs `model_dump(exclude_unset)`; delete is **soft** (`deleted_at`). Each action is **audited**
  via `FILE_UPDATED` with a human summary ("Edited/Added/Removed a stated …").
- **Per-resource tenant scoping** (`services/stated_financials.py`): `get_stated_{liability,asset,
  income,employer}_for_company` join the row → [borrower →] file → company; a miss returns `None` →
  **404** (anti-enumeration). `company_id` always from `current_user`.
- **Input/Public schemas** (`schemas/stated_financials.py`): per-kind `*Input` (all fields optional);
  the read now carries each row's `id` and returns employers as objects
  (`StatedEmployerPublic{id, employer_name, is_current}`) so the editor can target rows.

**Frontend**

- **`useStatedFinancialsEdit(fileId)`** (`lib/api/mismo.ts`): one hook owning all edit mutations —
  `updateRow`/`deleteRow` (PATCH/DELETE by kind+id), `addLiability`/`addAsset`/`addIncome`/
  `addEmployer`, and `updateLoanTerms` (PATCH the file). Every mutation invalidates
  `statedFinancialsQueryKey(fileId)` (plus `["loan-file", fileId]` for terms) so the display re-reads.
- **`StatedFinancialsEditor`** (`components/file/overview/stated-financials-editor.tsx`): inline edit
  mode. A single generic `EditableRow` drives all four kinds from a `FieldDef[]` config; Save sends
  **only changed fields** (diffed vs. initial) and an emptied field as `null` (not `""`); per-group
  **Add** appends a blank row, **Remove** soft-deletes; stated loan terms are one PATCH on the file.
  Errors → LP-46 toast; pending → LP-47 spinner + disabled Save.
- **`StatedFinancialsSection`**: an Edit/Done toggle in the card header swaps the read-only display
  for the editor; employer rendering updated for the new object shape; the warnings banner now points
  to the editor.
- **Types** (`lib/types/stated-financials.ts`): row `id`s added; `StatedEmployer` object type;
  `StatedBorrower.employers` now `StatedEmployer[]`.

## Safety

- **Audited** — every stated edit writes a `FILE_UPDATED` activity entry (reused the existing enum to
  avoid a migration for a within-file edit).
- **Tenant-scoped** — cross-company edit/add/delete → 404.
- **SSN** — replaced via the encrypted re-enter path; re-encrypted, masked in the response, never
  edited masked-in-place, never echoed or logged.
- **Validated** — bad amount → 422 → friendly LP-46 message.

## Assumptions / Decisions

- Reuse `FILE_UPDATED` rather than add an `ActivityType` value (no migration for a within-file edit);
  edit provenance is the summary text, not a typed field-level diff — a finer audit is a later change.
- UI scope (per product call) is the **stated financials + stated loan terms** editor; MISMO core
  fields beyond loan terms are backend-ready through the extended PATCH schemas (e.g. SSN replace)
  and surfaced by Epic-4 surfaces / future UI, not duplicated here.
- Add appends a blank row to fill (then Save PATCHes it) — keeps one row component for both new and
  existing rows; acceptable given all stated `*Input` fields are optional.

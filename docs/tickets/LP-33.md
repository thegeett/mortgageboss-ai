## Summary

Built the **file workspace shell** — the tabbed page for a single loan file — as a Next.js
nested layout: a persistent header (borrower / `display_id` / status / dates) + six-tab
navigation, the Overview as the default tab, and clearly-labeled "coming in Phase X"
placeholders for the rest. This is the destination of dashboard rows and the intake
redirect (which were 404ing). **Shell/structure only** — tab content comes in later tickets.

## The Shell Structure

`app/(protected)/loan-files/[id]/`:
- **`layout.tsx`** (client) — `useLoanFile(id)` → renders `FileHeader` + `FileTabs` +
  `{children}`. The header/tabs persist; only the tab content changes. `404` →
  `FileError` ("File not found").
- **`page.tsx`** — Overview (default tab), placeholder for now.
- **`documents/` `verification/` `communication/` `conditions/` `lender-package/`** —
  placeholder tab pages.

`components/file/` — `FileHeader` (skeleton when loading), `FileTabs` (route-based links,
`aria-current` from `usePathname`), `TabPlaceholder` (the "coming in Phase X" card),
`FileError` (not-found / error). `lib/loan-files/tabs.ts` — `FILE_TABS` config +
`activeTabKey`/`tabHref`. `useLoanFile` + `LoanFileDetail` added to the LP-31 data layer.

**Adding real tab content** (LP-34+): replace a tab's placeholder `page.tsx`; it fetches the
same `["loan-file", id]` query (deduped) and renders into the existing shell.

## Decisions Made

- **ADR-108** — file detail as a nested layout with a persistent header + route-based tabs.
- **ADR-109** — show all file tabs now with clearly-labeled placeholders.

## Assumptions

- **Overview content is LP-34**; documents Epic 5; later tabs in their phases.
- **No editing / file actions** here (the header *displays* status; doesn't change it).
- **`display_id` in the URL** (the API accepts UUID or display_id; dashboard + intake
  already navigate by display_id).
- **No backend changes** — the LP-28 detail API already provides what's needed.

## Verification Performed

- **Frontend:** `pnpm lint` (Biome) clean, `pnpm typecheck` clean, `pnpm build` succeeds
  (all six `/loan-files/[id]/*` routes compile), `pnpm test` → **32 passed** (incl. 3 new

- **Headless browser (Playwright, mocked detail, one-off — tooling reverted):** 16

## Notes

- The `aria-current` is correctly present on **both** the LP-27 sidebar "Loan Files" item
  (we're under `/loan-files`) and the active file tab — both are right; the test scopes its
  assertion to the file tab nav.
- Tabs are route links (not ARIA `tablist`/`tabpanel`) — the correct semantics when each tab
  is its own URL.
- `StatusBadge` moved from `components/dashboard/` to `components/status-badge.tsx` (neutral)
  and the dashboard import updated — one shared badge, reused by the file header.
- Playwright was a throwaway verification tool (already present, not re-added to
  `package.json`).

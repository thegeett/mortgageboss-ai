# LP-45 — Integration Test Suite for the API

- **Ticket:** LP-45 — API integration test suite
- **Epic:** Epic 6 — Testing, Polish & Phase 1 Completion
- **Status:** Completed
- **Date:** 2026-06-12

## Summary

Opens Epic 6 with a broad **integration** suite that drives the whole API end-to-end through the
**real stack** and verifies tenant isolation **systematically**. Where the per-ticket unit tests
mock the AI and test models/services in isolation, these tests exercise the seams: real HTTP →
routing → DI → auth → services → tenant-scoping → DB → storage → response. Only the AI and Celery
`.delay` are mocked. Backend only.

## What Was Built

**Reusable infrastructure (`backend/tests/integration/`)**

- **`conftest.py`** — composable fixtures, reusing the root `test_engine` (hierarchical conftest,
  not duplicated):
  - `db` — commit-safe savepoint session (endpoint `commit()`s land on SAVEPOINTs inside one
    outer transaction, rolled back per test → no leakage).
  - `client` — unauthenticated httpx `AsyncClient` via `ASGITransport` against the real app,
    sharing the test `db` (via `get_db` override).
  - `auth_client` — the Company A actor with a real Bearer token attached.
  - `company_a`/`user_a` and `company_b`/`user_b` — two tenants (actor + the company whose
    resources A must never reach).
  - `storage_tmp` (autouse) — local storage pointed at a temp dir, so uploads/downloads
    round-trip real bytes without touching real storage.
  - `mock_dispatch` (autouse) — stubs `process_document.delay` / `reprocess_document.delay`; the
    `Dispatch` handle exposes `.process` / `.reprocess` for "was it enqueued?" assertions.
- **`factories.py`** — minimal async factories (`make_company`, `make_user`, `token_for`,
  `make_loan_file`, `make_borrower`, `make_property`, `make_needs_item`, `make_lender`,
  `make_document` [stores real bytes], `make_extraction`), each flushing (never committing).

**End-to-end flow tests (real stack; AI + dispatch mocked)**

- `test_auth_flow` — login → token works on `/auth/me`; no token / bad token → `401`. (Phase 1 has
  no public *register* endpoint.)
- `test_loan_file_flow` — create (201) → list (appears) → get (200) → update (200, persists) →
  soft-delete (204) → gone from list / `404` on get.
- `test_document_flow` — upload (multipart PDF, 201, `pending`, `process.delay` asserted) → list →
  get (`current_extraction` slot) → download (bytes match) → soft-delete; validation
  (disallowed type / content spoof → 415); enqueue-failure resilience (still 201).
- `test_borrower_property_flow` — borrower CRUD (SSN masked on write) and property CRUD incl. the
  singleton `409`.
- `test_reads_and_override_flow` — needs / activity / borrowers / property sub-reads + the
  company lender list; the override flow (PATCH type → 200, category re-derived,
  `reprocess.delay` called, `document_type_overridden` activity surfaced) + enqueue-failure
  resilience.
- `test_pipeline_smoke` — the processing core over the real DB with only classify + extractor
  mocked → `COMPLETED` + a current `Extraction`. (The full lifecycle matrix stays in
  `tests/tasks/test_document_processing.py`; not duplicated.)

**Contract / leak checks (`test_contracts_leaks`)**

- No `storage_path` (documents), no `inbox_token` (loan files), no raw `ssn` (borrowers —
  `masked_ssn` only), no unmasked `account_number` (extraction — only `account_number_masked`).
- Status codes (201/200/204/404/422) and Pydantic response-model validation
  (`LoanFileDetail`, `DocumentDetailResponse`).

## Tenant Isolation Coverage (the security goal — COMPLETE)

`test_tenant_isolation.py`. A Company A actor → Company B resource → **404** (never the data,
never 403); lists never leak. Every company-scoped route enumerated and covered:

| Route | Cross-company result |
|---|---|
| `GET /loan-files/{id}` | 404 |
| `PATCH /loan-files/{id}` | 404 |
| `DELETE /loan-files/{id}` | 404 |
| `GET /loan-files` (list) | excludes B's files |
| `GET /loan-files/{id}/borrowers` | 404 |
| `POST /loan-files/{id}/borrowers` | 404 |
| `GET /loan-files/{id}/property` | 404 |
| `POST /loan-files/{id}/property` | 404 |
| `GET /loan-files/{id}/needs` | 404 |
| `GET /loan-files/{id}/activity` | 404 |
| `GET /loan-files/{id}/documents` | 404 |
| `POST /loan-files/{id}/documents` | 404 |
| `GET /documents/{id}` | 404 |
| `PATCH /documents/{id}` (override) | 404 |
| `GET /documents/{id}/download` | 404 |
| `DELETE /documents/{id}` | 404 |
| `POST /dev/documents/{id}/extract-text-layer` | 404 |
| `GET /lenders` (list) | excludes B's lenders |

The nested routes (borrowers/property/needs/activity/documents) are gated by `ScopedLoanFile`,
which resolves the parent file scoped to the caller's company first — so the file gate is the
tenant boundary (a B file → 404 before any child is reached). The flat document routes are gated by
`get_document_for_company` (joins document→loan_file→company). **Auth boundary:** a compact pass
asserts every protected route family rejects a missing token with `401`. (No role-gated business
route exists in Phase 1 — `require_role`'s `403` is unit-tested in `tests/api/test_dependencies.py`
— so there is no mounted `403` route to exercise here.)

## Coverage Achieved

Measured with `uv run pytest --cov=app --cov-report=term-missing`.

- **Overall: 93% of `app/`** (target was ~70%).
- **API layer** (the focus): `activity` 100%, `auth` 94%, `borrowers` 100%, `dev` 100%,
  `documents` 99%, `lenders` 100%, `loan_files` 100%, `needs` 100%, `property` 100%;
  `dependencies` 81% (the unused `OptionalCurrentUser` path).
- Remaining gaps are deliberate: `app/main.py` 76% (startup/lifespan), `document_processing.py`
  80% (deep `_mark_failed` rollback-retry branch, covered structurally in `tests/tasks`),
  `scripts/seed_dev.py` 0% (a dev script), and one HTTP-unreachable defensive guard in
  `documents.py:127`. No trivial padding was added.

## Bugs Surfaced & Fixed

1. **Coverage was silently under-measuring the API layer (~20pp).** There was no `[tool.coverage]`
   config, so coverage used the default thread-only tracer. SQLAlchemy's async engine runs DB work
   inside greenlet-spawned contexts (`greenlet_spawn`); the tracer dropped every line executed
   during request handling — handlers that demonstrably ran (tests got real `200`/`201`) showed as
   *uncovered* (e.g. `loan_files.py` measured 60%). **Fix:** added `concurrency = ["greenlet",
   "thread"]` (+ `source`/report config) in `pyproject.toml`. Overall measured coverage rose 89% →
   **93%** and the API layer to ~99–100%. This is a *measurement* defect (ADR-153), exactly what
   the coverage AC exists to expose.

No product-code tenant-isolation holes or leaked sensitive fields were found — the `ScopedLoanFile`
/ `get_document_for_company` gates and the SSN/`inbox_token`/`storage_path`/account-number masking
held across **every** enumerated route. That the systematic pass found no scoping bug is the
intended positive result: isolation was applied consistently in Epics 2–5.

## Decisions Made

- **ADR-152** — Integration test strategy: real stack, mock only AI + Celery dispatch; systematic
  tenant isolation.
- **ADR-153** — Coverage must trace SQLAlchemy's greenlet context (`concurrency`).

## Assumptions

- Frontend is tested separately (this is API integration only).
- AI and the Celery worker are mocked (no real key, no real broker).
- These complement, not replace, the unit suites.
- ~70% overall is a guide; **complete** tenant-isolation coverage matters more (achieved).

# LP-38 ‚Äî Document Classification Module

- **Ticket:** LP-38 ‚Äî Document classification module (classify_document, typed result, prompt-as-file, graceful failure)
- **Epic:** Epic 5 ‚Äî Document Upload & Processing
- **Status:** Completed
- **Date:** 2026-06-11

## Summary

Built the document **classification** module: given a document's already-extracted text,
`classify_document(text)` returns what KIND of document it is (`pay_stub`, `bank_statement`,
`w2`, ‚Ä¶) with a confidence and reasoning. First real use of the LP-37 AI wrapper and the
first act of the system "understanding" a document; classification routes extraction (LP-39
is type-specific). Backend only; tests mock the wrapper (no real calls, no key).

## What Was Built

- **`ClassificationResult`** (Pydantic, `app/ai/classification.py`) ‚Äî `document_type: str`,
  `confidence: float` (`ge=0, le=1`), `reasoning: str`, plus the
  `ClassificationResult.unknown(reason)` graceful fallback.
- **Prompt file** ‚Äî `app/ai/prompts/classification/document_classifier.txt`, a clearly
  marked **STARTER** that instructs the model to return JSON
  (`document_type` / `confidence` / `reasoning`) and use `unknown` when unsure.
- **Prompt loader** ‚Äî `app/ai/prompt_loader.py::load_prompt(relative_path)`: reads a prompt
  from the prompts dir (CWD-independent, path-escape-checked, `@cache`d). The pattern
  extraction (LP-39) reuses.
- **`classify_document(text)`** ‚Äî empty-text short-circuit, prompt load, LP-37 `complete`
  call with the classification model, defensive JSON parse, graceful `unknown` on any
  failure, metadata-only logging.
- **`_parse_classification_json` / `_extract_json_object` / `_coerce_confidence`** ‚Äî the
  defensive parser.

## The Contract

`classify_document(text: str) -> ClassificationResult` ‚Äî always returns a typed result,
never raises. `document_type` is a **flexible string** (LP-15), not an enum. This module
**returns** a result; persisting it onto the `Document` is the pipeline's job (LP-42), which
treats `unknown` / low-confidence as `NEEDS_REVIEW`.

The text is sent as the **user** message; the file-based prompt is the **system** prompt
(keeps the instructions and the untrusted document content separate).

## Prompt-as-File (a POC asset)

The prompt is loaded from the file, never hardcoded in Python ‚Äî so swapping it is a content
edit. **The shipped prompt is a STARTER** (clearly marked at the top of the file): the real,
tuned classification prompt is a POC asset and must be pasted into
`document_classifier.txt`, replacing the starter. No code change is needed to swap it.

## Graceful Failure

`classify_document` **never raises**. Failure paths all return
`ClassificationResult.unknown(...)`:

| Situation | Result |
| --- | --- |
| empty / whitespace / `< 10` chars | `unknown("empty or insufficient text")` ‚Äî **no API call** |
| `AIClientError` from the wrapper | `unknown("AI call failed")` |
| unparseable / non-JSON response | `unknown("could not parse classification")` |
| valid but low confidence | returned **as-is** (the pipeline flags review, not this module) |

The defensive parser extracts the first balanced `{...}` object (tolerating ```` ```json ````
fences and surrounding prose), clamps `confidence` to `[0, 1]` (coercing numeric strings,
rejecting bools/junk ‚Üí `0.0`), and treats a missing/empty `document_type` as `unknown`.

## Privacy

The document text and the model's raw response carry borrower PII and are **never** logged.
Only metadata is logged here ‚Äî `classification_succeeded` carries `document_type` +
`confidence`; the failure logs carry no content. The wrapper (LP-37) logs call metadata
(tokens/latency), also content-free. A test seeds PII into the text and the response and
asserts neither appears in captured logs.

## Uses the LP-37 Wrapper

Calls `app.ai.client.complete(...)` (not the SDK) with
`settings.anthropic_model_classification` (a cheaper Haiku-class model intended for
classification ‚Äî a `TODO`-marked placeholder to verify) and a small `max_tokens` (512, the
output is a tiny JSON object). Token usage flows back via the wrapper for cost tracking.

## Decisions Made

- **ADR-120** ‚Äî classification returns a typed result; type is a flexible string.
- **ADR-121** ‚Äî prompts stored as files, loaded at runtime (starting with classification).
- **ADR-122** ‚Äî graceful failure: classification never crashes the pipeline.

> **ADR numbering:** LP-37 used 117‚Äì119; the next free numbers were **120‚Äì122**.

## Assumptions

- **PDF text extraction** is LP-40 (this takes text, not bytes/PDFs); **field extraction**
  is LP-39; the **pipeline that calls this and persists the result** is LP-42; the **full
  ~100-type taxonomy** is Phase 2.
- The shipped prompt is a **starter to replace with the POC prompt**.
- Reused the existing `anthropic_model_classification` setting (the spec's
  `ai_classification_model`) ‚Äî both name the classification model.
- Tests **mock** the wrapper ‚Äî no real calls, no key (CI safe).

## Verification Performed

`uv run pytest` ‚Üí **397 passed** (18 new across classification + prompt loader), and
`tests/ai/` passes **without a real key** (the wrapper is mocked). ruff check + format,
`mypy app/` (strict), pre-commit all clean.

- **Success** ‚Äî well-formed JSON parses to the result; uses the classification model; sends
  the text as the user message and the prompt as `system`.
- **Fenced JSON** ‚Äî a ```` ```json ```` / preambled response still parses.
- **Low confidence** ‚Äî returned as-is (`0.3`), not overridden.
- **Malformed** ‚Üí `unknown` (no crash); **AIClientError** ‚Üí `unknown` (not propagated);
  **empty/short text** ‚Üí `unknown` **without** calling the API (asserted `call_count == 0`).
- **Confidence clamp** ‚Äî `1.5 ‚Üí 1.0`, `-0.2 ‚Üí 0.0`, `"0.42" ‚Üí 0.42`, junk/bool ‚Üí `0.0`.
- **Privacy** ‚Äî PII text and raw response content are absent from captured logs; the success
  log carries only `document_type` + `confidence`.
- **Loader** ‚Äî reads the file, CWD-independent, rejects path traversal, raises on a missing
  file.

## Summary

Established the database migration foundation for Epic 2. Created the SQLAlchemy
declarative `Base` with a constraint **naming convention**, reusable mixins
(`TimestampMixin`, `SoftDeleteMixin`, `UUIDMixin`) and a `utcnow()` helper;
initialized **Alembic** with the async template and wired `env.py` to the app's
settings and metadata; and authored the first migration enabling the `pgcrypto`
and `uuid-ossp` PostgreSQL extensions. Verified the full pipeline — upgrade,
downgrade, current, history — and that autogenerate detects models (with the
naming convention applied). No business tables were created; this ticket is
purely foundational.

Implemented in three reviewed stages: (1) base + mixins, (2) Alembic async
config, (3) first migration + verification.

## What Was Built

### Base classes and mixins (`backend/app/models/base.py`)

| Class / helper | Purpose |
| --- | --- |
| `Base` | Declarative base; `MetaData` carries the constraint naming convention. |
| `TimestampMixin` | `created_at` + `updated_at`, tz-aware `timestamptz` (UTC); `updated_at` auto-bumps. |
| `SoftDeleteMixin` | `deleted_at` column + `is_deleted` property; no automatic global filtering. |
| `UUIDMixin` | `uuid4` UUID primary key. |
| `utcnow()` | Timezone-aware UTC `datetime` helper. |

`app/models/__init__.py` re-exports `Base`, the mixins, and `utcnow` so Alembic
autogenerate can discover all tables once concrete models are imported there.

### Alembic configuration

- **`alembic.ini`** — removed the hardcoded `sqlalchemy.url` (env.py supplies it
  from settings); set a timestamped `file_template`.
- **`alembic/env.py`** — async configuration: injects
  `settings.database_url` (asyncpg), `target_metadata = Base.metadata`, runs
  migrations via `async_engine_from_config` + `connection.run_sync`, with
  `compare_type` / `compare_server_default` enabled. Supports online and offline
  modes.

### First migration

`..._enable_postgres_extensions.py` (hand-written): `CREATE EXTENSION IF NOT
EXISTS pgcrypto` and `"uuid-ossp"`; downgrade drops both.

## Migration Commands

| Action | Command |
| --- | --- |
| Autogenerated migration | `uv run alembic revision --autogenerate -m "..."` |
| Empty migration | `uv run alembic revision -m "..."` |
| Apply | `uv run alembic upgrade head` |
| Roll back one | `uv run alembic downgrade -1` |
| Roll back all | `uv run alembic downgrade base` |
| Current revision | `uv run alembic current` |
| History | `uv run alembic history` |

Full reference: [`docs/database.md`](../database.md).

## Decisions Made

- **ADR-031** — Alembic for database migrations (async config).
- **ADR-032** — Constraint naming convention (set before any tables exist).
- **ADR-033** — Timezone-aware timestamps in UTC.
- **ADR-034** — UUID primary keys (with the loan_files exception).
- **ADR-035** — pgcrypto extension for encryption.

## Assumptions

- PostgreSQL 16 is running locally (Docker Compose); dev DB `mortgageboss_dev`,
  user `mortgageboss`.
- The configured `database_url` uses the `postgresql+asyncpg://` driver, which
  the async migration engine requires.
- `uuid-ossp` is enabled for optionality; UUIDs are currently generated in
  Python via `uuid4`.

## Verification Performed

- `uv run alembic upgrade head` → migration applied; `pgcrypto` and `uuid-ossp`
  present in `pg_extension`.
- `uv run alembic downgrade base` → both extensions dropped (only built-in
  `plpgsql` remained); re-`upgrade head` restored them.
- `uv run alembic current` shows `f0934055772d (head)`; `uv run alembic history`
  shows `<base> -> f0934055772d`.
- **Autogenerate** tested with a throwaway model: detected the added table and
  applied the naming convention (`pk_throwaway_widget`). The throwaway model,
  its `__init__` import, and the generated migration were then **deleted** — no
  leftovers in `app/models` or `alembic/versions`.
- `uv run ruff check`, `ruff format --check`, `uv run mypy app/`, `uv run pytest`
  all pass.

## Notes

- The **naming convention is foundational** — it propagates to every constraint
  in every future table and cannot be changed cheaply later.
- **All timestamps are UTC**; always use `utcnow()`, never naive `datetime.now()`.
- `pgcrypto` is enabled now so encryption functions are ready when the borrower
  model (LP-14) adds encrypted columns; key management is deferred to Phase 7.
- Original phase-plan scoping had LP-10 as "core base and mixins"; that content
  was delivered here in LP-9, so LP-10 was repurposed to the remaining
  model-layer foundations (test infra, shared types, enums, helpers).

## Summary

~30 Conventional rules across **credit/DTI (13), property/appraisal (9), and documentation (8)** poured into
the LP-74 engine — CONTENT, not mechanism, the same shape + posture as LP-82. **GROUNDED STARTERS** researched
against the current Fannie Mae Selling Guide (retrieved 2026-06) with real B-section citations, every rule
`starter=True`, pending the domain expert's validation. The research **caught a major change** — the
credit-score minimum is now layered, not a flat 620. See **ADR-204**.

## The Headline Correction — credit score is LAYERED, not flat-620

The single most-repeated Conventional rule has materially changed. Encoded as the nuanced current state
(B3-5.1-01, marked recently-changed):
- **DU Version 12.0 (11/2025): no minimum** — DU relies on its own risk analysis (no rule; documented).
- **Manual underwriting: 620** — `conv.credit.min_score_manual`, **gated** to `underwriting.is_manual`.
- **Delivery floor: sub-620 ineligible for delivery** — `conv.credit.min_score_delivery_floor`, ungated.

A hardcoded "min 620 always" would be wrong — this is exactly why grounded-research-then-starter matters.

## The Researched Grounding (current Selling Guide, June 2026)

Layered credit score (B3-5.1-01, recently-changed); DU-50% / manual-36%→45% DTI (B3-6-02); 4-month appraisal
age (B4-1.2-04); property eligibility (B2-3-01). Eligibility-Matrix / DU items (value acceptance, occupancy/
units) are marked **driven-not-hardcoded**. Uncertain subsections carry `to_verify=True` (esp. the
derogatory-credit waiting periods — researched but the exact periods are flagged, not asserted as certain).

## Applicability Gating

A `RuleGate` (a fact path + a `Condition`) on each gated rule; the engine evaluates the rule only when the
gate holds. `underwriting.is_manual` gates the manual min-score + manual DTI ceiling; `property.is_condo` gates
the condo project-review doc. An absent gate fact → not-applicable (conservative — don't fire a manual-only
rule when the method is unknown).

## Cross-Links (confirmed, not duplicated)

  undisclosed-obligation finding (the AI surfaces it; applying it feeds the DTI recompute — the interlock

## Verification Performed

- **Structure** — 30 rules (8 credit + 5 DTI + 9 property + 8 doc), each investor/conventional, typed reads,
  threshold-as-data, severity, structured citation (`section` + `retrieved=2026-06`), `starter=True`.
- **Layered credit score** — manual rule gated, delivery-floor ungated, both B3-5.1-01, marked recently-changed.
- **Deterministic evaluation** — a manual file with a 610 score fires the manual min-score; a 48% manual DTI
  fires the manual ceiling (consuming the calc fact); a >4-month appraisal fires.
- **Applicability gating** — the manual-only rule does NOT fire for a DU file (or an unknown method); the
  condo-only review does NOT fire for a single-family, and DOES fire (review missing) for a condo.
- **Overlay-overrideability** — an overlay overrides `conv.credit.min_score_delivery_floor` (620→640) and
  `conv.property.appraisal_age` (4→3) by `rule_id`.
- **Test-file evaluation** — a file with a subject property + DTI inputs → the property promotions appear, the
  subject-property rule passes, the DTI fact is available, and the condo rule is not-applicable (single-family).
- **No regression** — backend **1262 passed**; `ruff`/`mypy --strict` clean. No migration.
